# Flutter 自适应与响应式设计 官方笔记

（严格对标文档结构，无额外扩展）

## 一、设计重要性

无论面向移动端还是Web端，应用都需要适配**屏幕尺寸、方向变化**，并最大化利用各平台特性。

## 二、Flutter 核心目标

基于**单一代码库**，开发在任意平台都拥有良好视觉与交互体验的应用。

## 三、适配场景

应用可运行在多种尺寸屏幕（手表、折叠屏、高清显示器），并兼容键盘、鼠标、触屏等多种输入设备。

## 四、响应式 vs 自适应（核心定义）

### 1. 核心区分

- 响应式设计（Responsive）：让UI适配可用空间，**调整设计元素布局**
- 自适应设计（Adaptive）：让UI在空间中可用，**选择合适的布局与输入设备**

### 2. 通俗解释

- 响应式：调整元素位置，贴合可用空间
- 自适应：选用适配的布局、输入方式，保证空间可用性

### 3. 示例

平板UI应使用底部导航，还是侧边栏导航。

### 4. 提示

两者概念常被合并统称，**自适应设计**通常可同时指代自适应与响应式。
