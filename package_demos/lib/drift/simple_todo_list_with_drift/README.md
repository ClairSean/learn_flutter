# 极简本地待办列表DEMO

## 核心定位

纯本地待办事项 Demo，仅实现基础增删改查，无网络、无复杂 UI、无高级状态管理。

## 技术栈

- 框架：Flutter 原生组件
- 语言：Dart
- 本地数据库：Drift (封装 SQLite，实现数据持久化)
- 使用provider进行状态管理

## 功能需求

**列表展示**：首页展示所有本地待办事项，分组显示已完成/未完成事项，未完成事项升序排序，已完成事项降序排序

**新增待办**：输入标题，添加一条新待办，可填写待办事项说明内容

**状态切换**：点击复选框，标记待办「已完成 / 未完成」

**删除待办**：长按 / 点击删除按钮，删除单条待办

## 数据持久化

- 关闭 APP、杀死进程、重启手机，数据不丢失

- 数据库仅本地存储，无云端同步

## 数据表设计

待办表（todos）：

- `id`：主键，自增数字（唯一标识）
- `title`：待办标题（文本，必填，长度1-32）
- `content`：待办事项说明内容（文本，可为空字符串，长度0-1000）
- `isCompleted`：完成状态（布尔值，默认 false）
- `createdAt`：创建时间（自动生成）

## 架构

使用flutter官方建议MVVM架构，合理组织各部分代码和文件
