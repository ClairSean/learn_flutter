# 数据库迁移

安全修改数据库结构的工具与 API

Drift 通过严格的数据库结构（Schema）保证查询类型安全。如需修改数据库结构，**必须编写数据库迁移脚本**。Drift 提供了完整的 API、命令行工具以及测试工具，让编写、验证数据库迁移的过程更简单、更稳定可靠。

---

## 引导式迁移（Guided Migrations）

Drift 内置工具可以帮你逐步编写、测试迁移脚本。支持增量修改数据库结构，同时配套测试工具可以验证迁移逻辑是否正确。

### 配置

想要使用 `make-migrations` 命令，你需要在项目的 `build.yaml` 文件中配置数据库文件路径：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          databases:
            # 必填：数据库名称 + Dart 文件路径
            my_database: lib/database.dart

            # 可选：配置多个数据库
            another_db: lib/database2.dart
```

你还可以自定义测试文件、数据库结构文件的存放目录。
Drift 会为每一个数据库版本生成单独的结构文件，用于对比新旧结构、自动生成迁移代码。

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          # 测试文件目录（默认值）
          test_dir: test/drift/

          # 数据库结构文件目录（默认值）
          schema_dir: drift_schemas/
```

### 使用方法

1. 在修改初始数据库结构**之前**，先执行命令生成初始结构文件：

```bash
dart run drift_dev make-migrations
```

2. 保存初始结构后，就可以自由修改数据库表结构。
3. 修改完成后，在数据库类中升级 `schemaVersion` 版本号，再次执行命令：

```bash
dart run drift_dev make-migrations
```

该命令会自动生成以下文件：

1. 与数据库类同级的**分步迁移文件**，用于增量编写迁移逻辑；
2. 迁移脚本的**测试文件**，编写完成后运行测试即可验证迁移是否正确；
3. 测试文件会自带第一条迁移的数据完整性示例测试。

遇到问题可以随时在官方发起讨论。

### 示例

修改完数据库结构、执行生成命令后，就可以编写迁移逻辑。
如果你的数据库定义在 `database.dart`，Drift 会在同级生成 `database.steps.dart` 文件。
该文件包含所有历史版本结构快照，大幅简化迁移编写难度：

```dart
import 'package:drift/drift.dart';

import 'database.steps.dart';

part 'database.g.dart';

@DriftDatabase(...)
class MyDatabase extends _$MyDatabase {
  MyDatabase(super.e);

  // 表结构已修改，升级版本号
  @override
  int get schemaVersion => 2;

  @override
  MigrationStrategy get migration {
    return MigrationStrategy(
      onUpgrade: _schemaUpgrade,
    );
  }
}

extension Migrations on GeneratedDatabase {
  // 将 stepByStep 抽离为静态字段/方法，避免意外引用当前最新数据库结构
  // 保证每一步迁移都能精确对应历史版本快照
  OnUpgrade get _schemaUpgrade => stepByStep(
    from1To2: (m, schema) async {
      // 从版本1升级到版本2：创建 groups 表
      await m.createTable(schema.groups);
    },
  );
}
```

完整示例可查看 Drift 官方仓库。

## 切换至引导式迁移

如果你之前已经在使用原生结构工具编写迁移，可以按以下步骤切换到 `make-migrations`：

1. 执行命令生成初始结构文件；
2. 将所有旧结构文件移动到数据库对应的结构目录；
3. 再次执行命令，自动生成分步迁移文件和测试文件。

---

## 开发期间的临时处理

开发阶段你会频繁修改表结构，暂时不想编写迁移脚本时：
可以直接删除应用数据、重新安装应用，数据库会被清空并重新创建所有表。

> 注意：Android 系统可能会备份数据库文件，单纯卸载应用不一定删除数据库，重装时会恢复备份数据。
> 你也可以配置每次打开应用都删除并重建所有表。

## 手动迁移（Manual Migrations）

⚠️ **警告**：手动编写迁移极易出错，可能造成数据丢失。
官方推荐使用 `make-migrations` 命令自动生成迁移和测试脚本。

在数据库类升级 `schemaVersion` 版本号后，Drift 提供原生迁移 API 逐步执行结构修改。只需重写 `migration` getter 即可。

示例：
你先在待办事项表新增截止日期字段（版本2），之后又新增优先级字段（版本3）：

```dart
class Todos extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get title => text().withLength(min: 6, max: 10)();
  TextColumn get content => text().named('body')();
  IntColumn get category => integer().nullable()();
  DateTimeColumn get dueDate =>
      dateTime().nullable()(); // v2 新增：截止日期
  IntColumn get priority => integer().nullable()(); // v3 新增：优先级
}
```

数据库类迁移配置：

```dart
// 结构已更新，升级版本号
@override
int get schemaVersion => 3;

@override
MigrationStrategy get migration {
  return MigrationStrategy(
    // 全新安装数据库：直接创建所有表
    onCreate: (Migrator m) async {
      await m.createAll();
    },
    // 升级数据库：按旧版本逐步迁移
    onUpgrade: (Migrator m, int from, int to) async {
      // 旧版本 < 2：执行1→2迁移，新增截止日期
      if (from < 2) {
        await m.addColumn(todos, todos.dueDate);
      }
      // 旧版本 < 3：执行2→3迁移，新增优先级
      if (from < 3) {
        await m.addColumn(todos, todos.priority);
      }
    },
  );
}
// 类其余代码保持不变
```

你还可以创建、删除单独数据表，完整操作参考 `Migrator` API 文档。

> 注意事项：
> 迁移回调中虽然可以使用 select/update/delete 等查询 API，但 Drift 始终以**最新版本结构**解析 SQL 和结果。
> 例如：新增列之前，不能查询该表；尽量避免在迁移回调中执行查询。

## 迁移完成后回调（Post-Migration callbacks）

`MigrationStrategy` 中的 `beforeOpen` 参数，可以在**数据库迁移完成、任何业务查询执行之前**初始化数据。

该回调每次打开数据库都会执行，无论是否执行了迁移。
可以通过 `details.hadUpgrade` / `details.wasCreated` 判断是否是全新安装/是否执行了升级：

```dart
beforeOpen: (details) async {
  // 全新创建数据库时初始化默认数据
  if (details.wasCreated) {
    final workId = await into(categories).insert(Category(description: '工作'));

    await into(todos).insert(TodoEntry(
          content: '第一条待办',
          category: null,
          targetDate: DateTime.now(),
      ));

    await into(todos).insert(
          TodoEntry(
            content: '重构持久化代码',
            category: workId,
            targetDate: DateTime.now().add(const Duration(days: 4)),
      ));
  }
},
```

你也可以在这里开启数据库配置，比如外键约束：

```dart
beforeOpen: (details) async {
  if (details.wasCreated) {
    // 初始化默认数据
  }
  // 开启外键约束
  await customStatement('PRAGMA foreign_keys = ON');
}
```
