# 运行时数据库结构自省

使用自动生成的数据表类，以反射方式检查数据库的结构

得益于 Drift 生成的**类型安全**数据表类，在 Dart 中编写 SQL 查询变得简单又安全。不过，这些查询通常是针对特定数据表编写的。尽管 Drift 支持数据表的继承，但在某些场景下，通过**反射**的方式访问数据表会更加便捷。幸运的是，Drift 生成的代码实现了专用接口，完全可以满足这个需求。

由于大部分 Drift 用户并不需要这个功能，本文主要提供实用示例，并链接相关类的官方文档。

举个例子：你可能有多个独立的数据表，它们都包含一个 `id` 列，并且希望通过 `id` 列筛选数据行。
如果针对单张表（比如入门示例中的 `Todos` 表）编写这个查询，代码非常直观：

```dart
Selectable<Todo> findTodoEntryById(int id) {
  return select(todos)..where((row) => row.id.equals(id));
}
```

但如果我们想把这个查询**通用化**，适配数据库中的所有表，该如何实现？下面的代码片段展示了具体方案（代码中的链接直接指向对应官方文档）：

```dart
extension FindById<Table extends HasResultSet, Row>
    on ResultSetImplementation<Table, Row> {
  Selectable<Row> findById(int id) {
    return select()..where((row) {
      final idColumn = columnsByName['id'];

      if (idColumn == null) {
        throw ArgumentError.value(
          this,
          'this',
          '必须是包含 id 列的数据表',
        );
      }

      if (idColumn.type != DriftSqlType.int) {
        throw ArgumentError('id 列必须为整数类型');
      }

      return idColumn.equals(id);
    });
  }
}
```

这段代码比单表查询复杂很多，我们逐行拆解核心逻辑：

1.  `FindById` 是作用于 `ResultSetImplementation` 的**扩展方法**。这个类是 Drift 为所有**数据表/视图**生成的基类，提供了检查数据库结构、将数据库原始数据映射为生成的数据类等实用方法。
    - `ResultSetImplementation` 接收两个泛型参数：原始数据表类、生成的数据行类。例如：你定义了 `class Todos extends Table`，Drift 会生成一个继承自 `Todos` 的类，同时实现 `ResultSetImplementation<Todos, Todo>`（`Todo` 是自动生成的数据类）。
    - `ResultSetImplementation` 有两个子类：`TableInfo`（数据表）和 `ViewInfo`（视图），分别混入到生成的数据表和视图类中。
    - `HasResultSet` 是 `Table` 和 `View` 的基类，这两个类用于在 Drift 中声明数据表和视图。
2.  `Selectable<Row>` 代表一个可执行查询，你可以调用 `get()`、`watch()`、`getSingle()`、`watchSingle()` 等方法执行查询。
3.  `findById` 中使用的 `select()` 扩展方法，可以**不依赖数据库实例**直接发起查询，仅需数据表对象即可。
4.  我们可以通过 `columnsByName` 根据 SQL 列名查找字段，这里我们要求表中必须存在一个整型列。
5.  `GeneratedColumn` 代表数据库中的一列，可以从列实例中读取约束、数据类型、默认值等信息。
    - 我们通过它校验数据表必须包含一个名为 `id` 的整型列。

调用这个扩展方法非常简单：

```dart
await myDatabase.todos.findById(3).getSingle();
```

将方法定义为扩展方法的一大优势是**类型推断完美生效**：在 `todos` 表上调用 `findById` 会直接返回 `Todo` 对象（该表对应的生成数据类）。

## 更新与插入操作

同样的思路也适用于构建**更新、删除、插入**语句（注意：视图是只读的，因此这些操作需要 `TableInfo`，而非 `ResultSetImplementation`）。

更新和插入操作使用 `Insertable` 对象，分别代表待更新/插入的部分数据行。针对已知数据表，我们会使用生成的类型安全 `Companion` 伴随类；而通过**数据库结构自省**，我们可以使用 `RawValuesInsertable` —— 这是一个通用的 `Insertable` 实现，基于**列名→值**的映射表构建数据。

下面的示例基于上文的代码，实现了**通用更新**：根据 `id` 筛选任意表，并更新其 `title` 列：

```dart
extension UpdateTitle on DatabaseConnectionUser {
  Future<Row?> updateTitle<T extends TableInfo<Table, Row>, Row>(
    T table,
    int id,
    String newTitle,
  ) async {
    final columnsByName = table.columnsByName;
    final stmt = update(table)
      ..where((tbl) {
        final idColumn = columnsByName['id'];

        if (idColumn == null) {
          throw ArgumentError.value(
            this,
            'this',
            '必须是包含 id 列的数据表',
          );
        }

        if (idColumn.type != DriftSqlType.int) {
          throw ArgumentError('Column `id` is not an integer');
        }

        return idColumn.equals(id);
      });

    final rows = await stmt.writeReturning(
      RawValuesInsertable({'title': Variable<String>(newTitle)}),
    );

    return rows.singleOrNull;
  }
}
```

在数据库或数据库访问类中，可以这样调用：

```dart
Future<Todo?> updateTodoTitle(int id, String newTitle) {
  return updateTitle(todos, id, newTitle);
}
```
