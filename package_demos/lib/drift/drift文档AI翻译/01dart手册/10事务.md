# 事务

原子化执行多条数据库语句

Drift 支持事务功能，允许多条语句**原子性执行**——在事务完成之前，其中的所有修改都不会对主数据库可见，确保数据操作的一致性。
要开启事务，只需在数据库或 DAO 实例上调用 `transaction` 方法，并传入一个函数作为参数，函数内的所有操作都会以事务形式执行。

以下示例实现了**删除分类**的功能：删除分类前，会将该分类下的所有待办事项移回默认分类：

```dart
Future<void> deleteCategory(Category category) {
  return transaction(() async {
    // 第一步：将受影响的待办事项移回默认分类
    await (update(todoItems)
          ..where((row) => row.category.equals(category.id)))
        .write(const TodoItemsCompanion(category: Value(null)));

    // 第二步：删除该分类
    await delete(categories).delete(category);
  });
}
```

## ⚠️ 事务使用的重要注意事项

使用事务时，必须牢记以下规则：

1. **必须等待所有数据库调用**
   事务内的所有查询都必须使用 `await` 等待执行。事务会在内部函数执行完成后结束，若未使用 `await`，部分查询可能会在事务关闭后继续执行，**导致数据丢失或运行时崩溃**。
   Drift 内置了运行时检查机制，检测到事务被关闭后仍被使用时会抛出异常。
   事务块内的所有异步操作都会占用事务，因此**不能在事务中创建定时器**或其他数据库操作（这些操作会尝试在事务结束后复用事务）。

2. **流查询的行为会发生变化**
   在事务回调内部创建的流查询，行为与普通流不同。详情请查看下一章节。

---

## 事务与流查询

- **事务外部创建的流**：会完美兼容事务内的修改。事务中对表的所有修改，**仅在事务完成后**才会通知流，保证数据始终一致，不会产生无效更新。
- **事务内部创建的流**：会**立即反映事务内的修改**，但会在事务完成时自动关闭。

该特性适合在事务内对流进行聚合操作（如调用 `first` 或 `fold`）。
**建议**：不要在事务外部监听事务内创建的流，否则会破坏事务的隔离性（流会暴露事务内部的中间状态）。

## 嵌套事务

从 Drift 2.0 版本开始，主流数据库实现均支持**嵌套事务**。
在事务块内部再次调用 `transaction` 方法（直接调用或通过方法间接调用），就会创建一个嵌套事务。

嵌套事务的执行规则：

1. 嵌套事务启动时，能看到**外层事务**在嵌套事务开始前的所有数据库状态；
2. 嵌套事务内的写入操作，**仅对当前嵌套事务可见**，外层事务和主数据库无法实时查看，流查询也不会更新；
3. 嵌套事务成功完成后，其修改会**原子性同步给外层事务**（外层事务的流查询会更新一次）；
4. 嵌套事务抛出异常时，会**自动回滚**（与普通事务一致）。外层事务可以捕获该异常，捕获后会恢复到嵌套事务启动前的状态；若未捕获，异常会向上抛出，同时回滚外层事务。

以下代码示例展示了嵌套事务的行为：

```dart
Future<void> nestedTransactions() async {
  await transaction(() async {
    // 插入第一个分类
    await into(categories).insert(CategoriesCompanion.insert(name: 'first'));

    // 嵌套事务 1
    await transaction(() async {
      // 此时能看到第一个分类
      await into(categories).insert(CategoriesCompanion.insert(name: 'second'));
      // 第二个分类仅在当前嵌套事务内可见
    });

    // 此时外层事务可以看到第二个分类

    try {
      // 嵌套事务 2
      await transaction(() async {
        // 能看到前两个分类
        await into(categories).insert(CategoriesCompanion.insert(name: 'third'));
        // 第三个分类仅在此嵌套事务内可见
        throw Exception('第二个嵌套事务中终止操作');
      });
    } on Exception {
      // 捕获异常，避免外层事务被回滚
    }

    // 此时第三个分类不可见，前两个分类保留
    // 事务状态恢复到嵌套事务2启动前
  });

  // 最终主数据库中只有两个分类
}
```

## 支持的数据库实现

嵌套事务需要底层数据库驱动的支持，以下主流 Drift 实现均兼容该功能：

- `package:drift/native.dart` 的 `NativeDatabase`
- `package:drift/wasm.dart` 的 `WasmDatabase`
- `package:drift/web.dart` 基于 sql.js 的 `WebDatabase`
- `package:drift_sqflite` 的 `SqfliteDatabase`

此外，如果服务端使用支持嵌套事务的数据库实现，**远程数据库连接**（隔离区、Web Worker 等）也支持嵌套事务。
