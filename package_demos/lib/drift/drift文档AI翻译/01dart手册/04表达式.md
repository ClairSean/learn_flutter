# Expressions（表达式）

深入了解可在 Dart 中编写的各类 SQL 表达式

表达式是数据库执行时会返回一个值的 SQL 代码片段。Drift 允许你在 Dart 中编写绝大多数表达式，然后自动转换为 SQL。表达式的应用场景非常广泛，例如 `where` 筛选条件就需要一个返回布尔值的表达式。

在大多数情况下，你编写的表达式都是由其他表达式组合而成的。**任意列名本身都是合法的表达式**，因此在大多数 `where` 子句中，你只需要将列名包装在各类比较逻辑中即可。

---

## 比较运算

所有表达式都可以通过 `equals` 方法与一个值进行比较。如果需要将一个表达式与另一个表达式对比，可以使用 `equalsExpr`。
对于数值和日期时间类型的表达式，还可以使用 `isSmallerThan`、`isSmallerOrEqual` 等多种方法进行比较：

```dart
// 查询所有腿数少于5条的动物
(select(animals)..where((a) => a.amountOfLegs.isSmallerThanValue(5))).get();

// 查询所有平均寿命短于腿数的动物（可怜的苍蝇）
(select(animals)
  ..where((a) => a.averageLivespan.isSmallerThan(a.amountOfLegs)));

Future<List<Animal>> findAnimalsByLegs(int legCount) {
  return (select(
    animals,
  )..where((a) => a.amountOfLegs.equals(legCount))).get();
}
```

## 布尔逻辑

你可以通过 Drift 提供的 `&`（与）、`|`（或）运算符和 `not` 方法，嵌套组合布尔表达式：

```dart
// 查询所有非哺乳动物且有4条腿的动物
select(animals)..where((a) => a.isMammal.not() & a.amountOfLegs.equals(4));

// 查询所有哺乳动物或有2条腿的动物
select(animals)..where((a) => a.isMammal | a.amountOfLegs.equals(2));
```

如果需要多个条件**全部满足**或**任一满足**，可以分别使用 `Expression.and` 和 `Expression.or`：

```dart
Expression.and([a.isMammal, a.amountOfLegs.equals(4)]);
```

## 算术运算

对于 `int` 和 `double` 类型的表达式，可以使用 `+`、`-`、`*`、`/` 运算符。
如果需要在 SQL 表达式和 Dart 值之间进行计算，将 Dart 值包装在 `Variable` 中即可：

```dart
Future<List<Product>> canBeBought(int amount, int budget) {
  return (select(products)..where((p) {
        // 总价 = 单价 × 数量
        final totalPrice = p.price * Variable(amount);
        return totalPrice.isSmallerOrEqualValue(budget);
      }))
      .get();
}
```

字符串表达式同样支持 `+` 运算符，与预期一致，它在 SQL 中执行**字符串拼接**。

对于整数值，可以使用 `~`、`bitwiseAnd`、`bitwiseOr` 执行**位运算**：

```dart
Expression<int> bitwiseMagic(Expression<int> a, Expression<int> b) {
  // 在 SQL 中生成 `~(a & b)`
  return ~(a.bitwiseAnd(b));
}
```

## BigInt 类型

SQLite 和 Dart 虚拟机使用 64 位整数，但编译为 JavaScript 的 Dart 应用不支持。因此，在 Web 端需要表示超大整数结果时，可以将表达式强制转换为 `BigInt`。

使用 `dartCast<BigInt>()` 可以让 Drift 将结果解析为 `BigInt`，**不会改变生成的 SQL**，Drift 为所有数据库统一使用 64 位整数类型。

示例：对于表达式 `(table.columnA * table.columnB).dartCast<BigInt>()`，即使 `columnA` 和 `columnB` 定义为普通整数，Drift 也会将结果以 `BigInt` 类型返回。

## 空值检查

若要检查表达式在 SQL 中是否为 `NULL`，可以使用 `isNull` 扩展方法：

```dart
final withoutCategories = select(todoItems)
  ..where((row) => row.category.isNull());
```

如果内部表达式结果为 `null`，该表达式会返回 `true`，否则返回 `false`。同理，`isNotNull` 作用相反。

若需要为 `null` 值提供兜底值，可以使用 `coalesce` 函数。它接收一组表达式，返回**第一个非空**的表达式结果：

```dart
// 对应 Dart 中的 ?? 运算符
final category = coalesce([todoItems.category, const Constant(1)]);
```

## 日期与时间

对于返回 `DateTime` 类型的列和表达式，可以使用 `year`、`month`、`day`、`hour`、`minute`、`second` 等 getter 提取日期的单个字段：

```dart
select(users).where((u) => u.birthDate.year.isSmallerThanValue(1950));
```

年份、月份等单个字段本身也是表达式，因此可以对它们使用运算符和比较逻辑。
若要获取当前日期/时间作为表达式，使用 Drift 提供的 `currentDate` 和 `currentDateAndTime` 常量。

你还可以使用 `+` 和 `-` 运算符为时间列增加/减少时长：

```dart
Future<void> increaseDueDates() async {
  final change = TodoItemsCompanion.custom(
    // 截止日期+1天
    dueDate: todoItems.dueDate + Duration(days: 1),
  );
  await update(todoItems).write(change);
}
```

对于更复杂的日期时间转换，`modify` 和 `modifyAll` 函数非常实用。例如，将所有待办事项的截止日期调整为下周一：

```dart
Future<void> moveDueDateToNextMonday() async {
  final change = TodoItemsCompanion.custom(
    dueDate: todoItems.dueDate.modify(
      DateTimeModifier.weekday(DateTime.monday),
    ),
  );
  await update(todoItems).write(change);
}
```

## IN 与 NOT IN 运算

通过 `isIn` 和 `isNotIn` 方法，可以检查表达式是否存在于值列表中：

```dart
select(animals)..where((a) => a.amountOfLegs.isIn([3, 7, 4, 2]));
```

`isNotIn` 的作用与 `isIn` 相反。

## 聚合函数（如计数、求和）

Drift 的 Dart API 内置了聚合函数。与普通函数不同，聚合函数**同时操作多行数据**：

- 默认情况下，它会将查询返回的所有行合并为一个值
- 也可以通过 `group by` 对结果分组，为每个分组执行聚合计算

### 极值比较

数值和日期时间表达式支持 `min`（最小值）和 `max`（最大值）方法，分别返回结果集中的最小/最大值。

### 算术聚合

支持 `avg`（平均值）、`sum`（求和）、`total`（总计）方法。例如，监听待办事项内容的平均长度：

```dart
Stream<double> averageItemLength() {
  final avgLength = todoItems.content.length.avg();
  // 使用 selectOnly 仅查询聚合结果，不查询表的原始列
  final query = selectOnly(todoItems)..addColumns([avgLength]);

  return query.map((row) => row.read(avgLength)!).watchSingle();
}
```

### 计数

统计分组内的行数非常实用。以下查询会统计每个分类关联的待办事项数量：

```dart
final amountOfTodos = todoItems.id.count();

final query = db.select(categories).join([
  innerJoin(
    todoItems,
    todoItems.category.equalsExp(categories.id),
    useColumns: false,
  ),
]);
query
  ..addColumns([amountOfTodos])
  ..groupBy([categories.id]);
```

- 若不想统计重复值，使用 `count(distinct: true)`
- 若仅统计满足条件的值，使用 `count` 的 `filter` 参数
- 若统计**所有行数**（而非单个列），使用顶层函数 `countAll()`

## group_concat 函数

`groupConcat` 函数可以将多个值拼接为单个字符串：

```dart
Stream<String> allTodoContent() {
  final allContent = todoItems.content.groupConcat();
  final query = selectOnly(todoItems)..addColumns([allContent]);

  return query.map((row) => row.read(allContent)!).watchSingle();
}
```

默认分隔符为**无空格的逗号**，可以通过 `groupConcat` 的 `separator` 参数自定义分隔符。

## 窗口函数

除了聚合表达式和 `groupBy`，Drift 还支持窗口函数。
与普通聚合函数将一组行合并为一个值不同，**窗口函数可以对与当前行相关的子集行执行聚合计算**，例如计算累计总值：

```dart
/// 返回所有待办事项，并为每个事项关联截止到当前行的标题总长度
Selectable<(TodoItem, int)> todosWithRunningLength() {
  final runningTitleLength = WindowFunctionExpression(
    todoItems.title.length.sum(),
    orderBy: [OrderingTerm.asc(todoItems.id)],
  );
  final query = select(todoItems).addColumns([runningTitleLength]);
  query.orderBy([OrderingTerm.asc(todoItems.id)]);

  return query.map((row) {
    return (row.readTable(todoItems)!, row.read(runningTitleLength)!);
  });
}
```

窗口函数的一个实用场景：为每行数据计算排序排名（无需实际返回所有行或按该列排序）：

```dart
/// 返回所有待办事项，并为每个事项标注按内容长度降序排列的名次（从1开始）
Selectable<(TodoItem, int)> todosWithLengthRanking() {
  final lengthRanking = WindowFunctionExpression(
    todoItems.id.count(),
    orderBy: [OrderingTerm.desc(todoItems.content.length)],
  );
  final query = select(todoItems).addColumns([lengthRanking]);

  return query.map((row) {
    return (row.readTable(todoItems)!, row.read(lengthRanking)!);
  });
}
```

## 子查询

Drift 对表达式中的子查询提供基础支持。

### 标量子查询

标量子查询是**仅返回一行一列**的查询语句，由于它只返回一个值，因此可以嵌入到其他查询中：

```dart
Future<List<TodoItem>> findTodosInCategory(String category) async {
  // 查询目标分类的ID
  final groupId = selectOnly(categories)
    ..addColumns([categories.id])
    ..where(categories.name.equals(category));

  final query = select(todoItems)
    ..where((row) => row.category.equalsExp(subqueryExpression(groupId)));
  return await query.get();
}
```

### isInQuery

与 `isIn`/`isNotIn` 类似，`isInQuery` 支持传入**子查询**而非直接传入值集合。
子查询必须仅返回一列，但可以返回多行，若值存在于子查询结果中则返回 `true`。

### Exists 存在性判断

`existsQuery` 和 `notExistsQuery` 函数用于检查子查询是否包含任意行。例如，查询空分类：

```dart
Future<List<Category>> emptyCategories() {
  final hasNoTodo = notExistsQuery(
    select(todoItems)..where((row) => row.category.equalsExp(categories.id)),
  );
  return (select(categories)..where((row) => hasNoTodo)).get();
}
```

### 完整子查询

Drift 还支持连接查询（JOIN）中的子查询，详情查看**连接查询**文档。

## 自定义表达式

若需要在 Dart 查询中嵌入自定义 SQL，可以使用 `CustomExpression` 类，通过 `sql` 参数编写自定义表达式：

```dart
// 定义：60天未登录的用户为非活跃用户
const inactive = CustomExpression<bool>(
  "julianday('now') - julianday(last_login) > 60",
);
select(users)..where((u) => inactive);
```

> **注意**：过度使用 `CustomExpression` 容易编写无效的查询。如果你因为 Drift 缺少某个功能而需要使用自定义表达式，建议提交 Issue 反馈；如果你更习惯直接写 SQL，也可以使用**编译型 SQL**，它是类型安全的。

当自定义表达式需要嵌入子表达式时，`CustomExpression` 会有局限性。更灵活的方案是**直接实现 `Expression` 接口**，完全控制 SQL 片段的生成。例如，实现 SQL 行值表达式：

```dart
/// 将行值 (`(1, 2, 3)`) 写入 SQL
/// 该表达式无法直接求值，仅作为子表达式使用，因此泛型绑定为 Never
final class RowValues extends Expression<Never> {
  final List<Expression> expressions;

  RowValues(this.expressions);

  @override
  Precedence get precedence => Precedence.primary;

  @override
  void writeInto(GenerationContext context) {
    context.buffer.write('(');

    for (final (i, expr) in expressions.indexed) {
      if (i != 0) context.buffer.write(', ');
      expr.writeInto(context);
    }

    context.buffer.write(')');
  }
}
```

使用方式：

```dart
void rowValuesUsage() {
  select(animals).where((row) {
    // 生成 SQL：(amount_of_legs, average_livespan) < (?, ?)
    return RowValues([
      row.amountOfLegs,
      row.averageLivespan,
    ]).isSmallerThan(RowValues([Variable(2), Variable(10)]));
  });
}
```
