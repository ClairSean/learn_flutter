# 流查询

在 Drift 中监听 SQL 查询

Drift 的核心特性之一是：**所有查询都可以转换为自动更新的数据流**。无论查询返回单行还是多行数据，无论查询单表还是多表连接，该功能都能正常使用。

---

## 基础用法

在 Drift 中，可执行查询由 `Selectable<T>` 接口表示，该接口包含以下方法：

- `Future<List<T>> get()`：执行一次查询，返回所有数据行
- `Future<T> getSingle()`：执行一次查询，确保仅返回单行数据
- `Future<T?> getSingleOrNull()`：与 `getSingle()` 类似，空结果集返回 `null`

以上每个方法都有对应的 `watch()` 方法，返回数据流：

- `Stream<List<T>> watch()`：监听查询，返回所有数据行
- `Stream<T> watchSingle()`：监听查询，确保每次执行仅返回单行数据
- `Stream<T?> watchSingleOrNull()`：与 `watchSingle()` 类似，空结果集返回 `null`

所有 Drift 查询构建 API 都会返回可监听的 `Selectable` 对象：

```dart
Selectable<TodoItem> allItemsAfter(DateTime min) {
  return select(todoItems)
    ..where((row) => row.createdAt.isBiggerThanValue(min));
}
```

无论使用哪种方式，都可以通过 `allItemsAfter(value).watch()` 创建数据流。
数据流是 Dart 中通用的编程组件，可被绝大多数框架直接使用：

- 在 Flutter 中，你可以使用 `StreamBuilder` 声明式监听数据流
- Riverpod 可以通过 `StreamProvider` 包装数据流，示例应用中也使用了该技术

所有 Drift 数据流在**开始监听后都会立即发送最新结果**（即使数据表从未发生变化，你也会收到一次数据快照，无需同时使用 `get()` 和 `watch()`）。

---

## 高级用法

除了监听查询本身，你还可以**直接监听数据表的更新事件**：

```dart
Future<void> listenForUpdates() async {
  final stream = tableUpdates(
    TableUpdateQuery.onTable(todoItems, limitUpdateKind: UpdateKind.update),
  );

  await for (final event in stream) {
    print('待办事项表发生更新：$event');
  }
}
```

完整的查询流功能由 Drift 内部实现，因此流更新是**启发式触发**，可能会比实际需要的触发次数更多。
你也可以**手动标记数据表已更新**：

```dart
void markUpdated() {
  notifyUpdates({TableUpdate.onTable(todoItems, kind: UpdateKind.insert)});
}
```

## 注意事项

虽然流查询可以自动获取查询的更新结果，但理解其原理和局限性非常重要。

### 工作原理

流查询在 Drift 中通过**启发式算法**实现：

1. 每个活跃的流都会追踪它所监听的数据表（信息来自查询构建器）
2. 每当通过 Drift API 执行插入、更新、删除操作时，关联的查询会重新执行

### 核心限制

1. **外部数据库操作不会触发更新**
   例如使用原生 SQLite 客户端直接操作数据库时，流查询不会自动更新。可以通过**手动触发更新**作为解决方案。
2. **流查询会频繁更新**
   由于无法仅针对特定行的更新进行过滤，流查询的触发次数通常会多于实际需要。这一般不会造成问题，但需要注意：流查询应尽量返回少量数据，且执行成本不宜过高。
