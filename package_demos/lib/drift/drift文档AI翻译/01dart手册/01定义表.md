# 数据表

定义你的数据库结构。

Drift 是面向关系型数据库开发的库，**数据表**是组织数据库最基础的单元。每张数据表对应一个独立业务实体/概念，同时定义了你存储数据的结构。
针对每张数据表，Drift 都会生成一套**类型安全的行数据类**，让你可以用高级 Dart 代码编写查询、更新操作，而不用手写原生 SQL。
本文档列出声明数据表、数据列时所有可用的配置选项。

## 定义数据表

所有 Drift 数据表都遵循统一的列定义规范：

1. 每张数据表定义为一个继承自 `Table` 的 Dart 类
2. 数据表类中，数据列定义为 `late final` 字段
3. 字段开头的方法（如 `integer()`）决定该列的数据类型

再回顾一遍入门示例里的数据表：

```dart
class TodoItems extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get title => text()();
  DateTimeColumn get createdAt => dateTime().nullable()();
}
```

**每一列结尾都必须额外加一对括号**，忘记书写 Drift 会给出编译警告。

注意：数据列默认**不可空**，调用 `.nullable()` 之后才允许存储 null 值。

上面代码定义了 `todo_items` 数据表，包含 `id`、`title`、`created_at` 三列。

该表对应的原生 SQL 建表语句：

```sql
CREATE TABLE todo_items (
  id INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT,
  title TEXT,
  created_at INTEGER -- Drift 默认将 dateTime() 列存储为 Unix 时间戳
);
```

### 技术说明

- 数据表名 `todo_items` 由 Dart 类名自动蛇形转换生成；可重写 `tableName` getter 自定义表名，详情见【数据表命名】章节
- `id` 列为自增整数，会**自动成为主键**，详情见【主键】章节
- 默认 `dateTime()` 列存储为 Unix 时间戳；如需存储为 ISO-8601 字符串，详见【DateTime 配置选项】

## 将数据表添加到数据库

在 `@DriftDatabase` 注解中填写数据表列表，即可完成注册：

```dart
@DriftDatabase(tables: [TodoItems])
class Database extends _$Database {
  Database(super.e);

  @override
  int get schemaVersion => 1;
}
```

**新增数据表后，必须重新运行代码生成器**：

```bash
dart run build_runner build
```

数据库**首次打开时**（一般是 App 首次启动），Drift 会自动创建所有已定义数据表。

如果数据库已经存在，Drift **不会自动修改原有表结构**。修改表结构的操作请参考【数据迁移】文档。

## 数据列类型

数据表中每一列都有固定类型，决定了它可以存储的值。Drift 提供大量内置列类型，满足绝大多数数据库开发需求。

| Dart 类型                    | Drift 列定义写法                      | SQL 类型              |
| ---------------------------- | ------------------------------------- | --------------------- |
| int                          | `late final age = integer()()`        | INTEGER               |
| BigInt（64位整数，查看原因） | `late final age = int64()()`          | INTEGER               |
| String                       | `late final name = text()()`          | TEXT                  |
| bool                         | `late final isAdmin = boolean()()`    | INTEGER（1=真，0=假） |
| double                       | `late final height = real()()`        | REAL                  |
| Uint8List                    | `late final image = blob()()`         | BLOB                  |
| DriftAny                     | `late final value = sqliteAny()()`    | ANY（用于严格数据表） |
| DateTime（查看配置选项）     | `late final createdAt = dateTime()()` | INTEGER 或 TEXT       |
| 自定义类型                   | 参考【类型转换器】文档                | 取决于转换后的类型    |
| 枚举                         | `intEnum()` / `textEnum()`            | INTEGER 或 TEXT       |
| PostgreSQL 类型              | 参考 PostgreSQL 文档                  | 取决于类型            |

除基础类型外，你还可以让列存储任意**可转换为内置类型**的数据，详情查看类型转换器文档。

## 主键

数据库每张表都必须有主键：用于唯一标识每一行数据的单个或多个列。

### 单自增主键

绝大多数数据表，使用单个自增整数列作为主键即可。

在 Drift 中，列定义里添加 `autoIncrement()` 即可声明自增主键，它会自动实现：

1. 将该列设为数据表**唯一主键**；不可在多列同时使用，也不可混合自增主键与自定义主键
2. 每新增一行数据，该列数值自动 +1

示例：

```dart
class Items extends Table {
  late final id = integer().autoIncrement()();
  late final title = text()();
}
```

#### 小技巧：使用 Mixin 混入复用公共列

你可以把多张表通用的列提取到 Dart Mixin 混入类中统一复用：

```dart
mixin TableMixin on Table {
  // 主键列
  late final id = integer().autoIncrement()();

  // 创建时间戳列
  late final createdAt = dateTime().withDefault(currentDateAndTime)();
}

class Posts extends Table with TableMixin {
  late final content = text()();
}
```

上面的 `Posts` 数据表会自动继承 `TableMixin` 中的 `id` 和 `createdAt` 列。

### 自定义主键

如果你需要用其他单个/多个列作为主键，在数据表类中重写 `primaryKey` getter 即可。

约束要求：

- 必须使用 `=>` 箭头语法定义，不支持大括号函数体
- 必须返回集合字面量，不能包含 `if`、`for`、扩展运算符等动态语法

```dart
class Profiles extends Table {
  late final email = text()();

  @override
  Set<Column<Object>> get primaryKey => {email};
}
```

以上代码将 `email` 列设置为主键。

# 定义数据列

在 Drift 中，数据列通过 `late final` 字段进行声明。字段值的开头部分决定了列的类型，你可以通过调用方法为列添加额外的修饰配置。**多个修饰符可以通过链式调用的方式作用于同一列**。

## 可空列

为列调用 `.nullable()` 后，该列将支持存储 `null` 值。
对于非空列，Drift 在插入数据行时会将相关参数标记为必填项：

```dart
late final age = integer().nullable()();
```

如果不调用 `nullable()`，`age` 列将成为必填列。尝试将已有行的该列设置为 `null` 会抛出异常。

## 默认值

有些列并非必须可空，但所有新数据行都可以共用一个合理的默认值。你无需在每次插入时都手动指定该值，直接将默认值配置在列上即可。

Drift 提供两种设置默认值的方式：

- `withDefault()`：为数据库 schema 中的列添加 `DEFAULT` 约束（在其他数据库框架中也被称为**服务端默认值**）。
- `clientDefault()`：不修改数据库 schema，而是在 Dart 层计算默认值，并自动将其注入 Drift 生成的插入语句中。

### withDefault()

将默认值以 **SQL 表达式** 的形式设置，由数据库自身执行。编写表达式的方法可参考【表达式】文档。
添加、移除或修改默认值属于**数据库结构变更**，需要谨慎处理（需编写迁移脚本）。

设置默认值的常见场景：为数据行添加创建时间列

```dart
late final creationTime = dateTime().withDefault(currentDateAndTime)();
```

即使是非空列，**配置了默认值后插入数据时无需传值**，数据库会自动使用默认值作为兜底。

### clientDefault()

与 `withDefault()` 类似，同样为列设置默认值。
区别在于：该默认值在 **Dart 代码层计算**，而非数据库层。

这意味着：添加、移除或修改默认值**不需要执行数据库迁移**。
但缺点是：该默认值仅在 Dart 代码中生效，在 Drift 之外直接操作数据库时不会生效。

```dart
late final useDarkMode = boolean().clientDefault(() => false)();
```

#### 推荐使用

绝大多数场景下，**推荐优先使用 `clientDefault()`**，因为它更灵活，且无需数据库迁移。

## 外键引用

在 Dart 表定义中，可以通过 `.references()` 方法为列声明**外键关联**：

```dart
class Albums extends Table {
  late final id = integer().autoIncrement()();
  late final name = text()();
  late final artist = integer().references(Artists, #id)();
}

class Artists extends Table {
  late final id = integer().autoIncrement()();
  late final name = text()();
}
```

`references` 的第一个参数指向要关联的目标表，第二个参数是目标表的列符号（Symbol）。

可选参数 `onUpdate` 和 `onDelete` 可以配置：当目标行被更新/删除时，当前行执行的联动操作。

---

⚠️ **注意**：在 sqlite3 中，外键约束**默认未启用**，需要执行 `PRAGMA foreign_keys = ON` 开启。
在 Drift 中，推荐在**数据迁移后的回调函数**中执行该语句。

---

## 额外校验约束

为列添加 **CHECK 检查约束**。在创建或更新数据行时，如果约束表达式结果为 `false`，数据库会抛出异常。
编写约束表达式的方法可参考【表达式】文档。

### 检查约束与数据迁移

如果现有数据不满足新添加的检查约束，**数据迁移会失败**。添加约束前请确保它与现有数据兼容。

### 示例

确保年龄大于等于 0：

```dart
late final Column<int> age = integer().check(age.isBiggerOrEqualValue(0))();
```

### 注意

要使用 `isBiggerOrEqualValue` 这类类型专属的表达式，**必须显式声明列的类型**。
如上例所示，`age` 列被显式定义为 `Column<int>`。

## 限制文本长度

为文本列设置最小/最大长度限制。
出于历史原因，该校验在 **Dart 层执行**，因此修改长度约束**不需要数据库迁移**。
如果需要更强的一致性校验，建议使用 CHECK 约束替代。

### 示例

确保名称非空且长度不超过 50 个字符：

```dart
late final name = text().withLength(min: 1, max: 50)();
```

## 生成列

使用 `.generatedAs()` 方法创建**生成列**，该列的值基于表内其他列计算得出。

与绝大多数数据库一致，Drift 支持两种生成列：

- **虚拟生成列**（默认）：值在每次查询时实时计算
- **存储型生成列**：值在插入/更新时计算并持久化存储

### 示例（虚拟生成列）

```dart
class Squares extends Table {
  late final length = integer()();
  late final width = integer()();
  // 面积列 = 长 × 宽，实时计算
  late final area = integer().generatedAs(length * width)();
}
```

# 唯一列

有些列虽然不属于主键，但仍需要存储**唯一值**。我们可以在数据库结构中强制启用唯一性约束，这同时能加速部分查询操作。
若要强制单列在所有行中保持唯一，在列定义中使用 `.unique()` 方法即可：

```dart
late final username = text().unique()();
```

若要强制**多列组合值唯一**，在数据表类中重写 `uniqueKeys` getter：

```dart
class Reservations extends Table {
  late final reservationId = integer().autoIncrement()();

  late final room = text()();
  late final onDay = dateTime()();

  @override
  List<Set<Column>> get uniqueKeys => [
    {room, onDay},
  ];
}
```

上面的示例会强制约束：**同一个房间不能在同一天被重复预订**。

### 主键无需额外配置

主键在每张表中天然唯一，因此主键列**无需**再添加唯一性约束。

# 索引

当**非主键、非唯一约束**的列经常在 `where` 子句中作为筛选条件时，可以使用**索引**来加速查询。
在大数据量表中，索引尤为重要：没有索引时，数据库引擎必须遍历每一行数据才能匹配筛选条件；
而索引会在后台创建并维护一个「索引值→数据行」的映射结构，让数据库无需扫描全表就能快速定位目标行。

---

使用 `@TableIndex` 注解创建索引，需要指定索引列与唯一索引名称。
将 `unique` 参数设为 `true`，可强制索引列的所有值保持唯一。

如需为一张表创建多个索引，添加多个 `@TableIndex` 注解即可。

### 注意

以下列会**自动创建索引**，无需手动定义：

1. 主键列
2. 唯一约束列
3. 外键约束的目标列

### 示例

如果用户表数据量很大，下面的索引会让基于用户名的查询效率大幅提升：

```dart
@TableIndex(name: 'user_name', columns: {#name})
class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text()();
}
```

如需为列指定排序模式，可以使用 `IndexedColumn` 实例。

---

### 基于 SQL 的索引

如果需要更复杂的索引配置（例如创建**局部索引**），也可以直接通过 SQL 语句定义：

```dart
@TableIndex.sql('''
  CREATE INDEX pending_orders ON orders (creation_time)
    WHERE status == 'pending';
''')
class Orders extends Table {
  IntColumn get id => integer().autoIncrement()();
  IntColumn get totalAmount => integer()();
  DateTimeColumn get creationTime => dateTime()();
  TextColumn get status => text()();
}
```

如你所料，Drift 会在编译期校验 `CREATE INDEX` 语句的合法性。

# 自定义约束

Drift 提供专用 API 来实现 SQL 中最常用的表约束与选项。
对于未直接支持的表约束，你可以通过嵌入 SQL 代码片段的方式在 Drift 中使用。

## 自定义列约束

类型化的列构造 API 已覆盖绝大多数列约束。
如果你需要更特殊的约束，可以使用 `.customConstraint()` 方法为列添加自定义 SQL 约束：

```dart
late final name = text().nullable().customConstraint('COLLATE BINARY')();
```

### 重要：自定义约束会覆盖 Drift 默认约束

添加 `customConstraint` 会**覆盖** Drift 自动添加的所有约束，最常见的是移除 `NOT NULL` 约束。
如果你需要保留列非空，必须手动添加该约束。

示例：

```dart
late final username = text().customConstraint('NOT NULL COLLATE BINARY')();
```

如果你忘记添加 `NOT NULL`，或尝试将自定义约束与不兼容的列选项混用，Drift 构建器会发出警告。

## 自定义表约束

你也可以在数据表类中重写 `tableConstraints` getter，为表本身添加自定义约束。

```dart
class TableWithCustomConstraints extends Table {
  late final foo = integer()();
  late final bar = integer()();

  @override
  List<String> get customConstraints => [
    'FOREIGN KEY (foo, bar) REFERENCES group_memberships ("group", user)',
  ];
}
```

### SQL 校验

无需担心语法错误或不支持的特性，Drift 会校验你提供的 SQL 语句，并在代码生成阶段抛出异常。

# 严格表（STRICT）与无 ROWID 表（WITHOUT ROWID）

SQLite 支持**严格表（strict tables）**，会对列应用更严格的类型校验规则。
Drift 当前默认未启用该选项，但未来大版本可能会默认开启。

重写 `isStrict` getter 即可将 Drift 定义的表设为严格表：

```dart
class Preferences extends Table {
  TextColumn get key => text()();
  AnyColumn get value => sqliteAny().nullable()();

  @override
  Set<Column<Object>>? get primaryKey => {key};

  @override
  bool get isStrict => true;
}
```

同理，重写 `withoutRowId` getter 可以创建**无 ROWID 表**。

# 高级结构选项

## 修改 SQL 命名

默认情况下，Drift 会将 Dart 的 getter 名称转为**蛇形命名法**，作为 SQL 中的列名。
例如 Dart 中的 `createdAt` 列，在 Drift 生成的建表语句中会变成 `created_at`。

使用 `.named()` 可以**显式指定列名**：

```dart
late final isAdmin = boolean().named('admin')();
```

### 仅需要修改大小写？

表名默认使用类名的蛇形形式，重写数据表类的 `tableName` getter 即可自定义表名：

```dart
class Products extends Table {
  @override
  String get tableName => 'product_table';
}
```

---

## BigInt 与 int64() 的使用场景

在 SQL 中，Drift 的 `integer()` 和 `int64()` 类型都会映射为存储 64 位整数的列（SQLite 中为 INTEGER）。
这意味着整数列与原生 Dart 的 `int` 行为一致。

但**编译为 JavaScript**时会出现问题：JavaScript 只有 64 位浮点数类型，无法精确表示超大数值。

因此，**需要编译为 JS、且整数列可能存储超大数值**的项目，Drift 提供了 `int64()` 类型：
它在 Dart 中以 `BigInt` 表示所有数字，避免 JS 兼容性问题。

---

## DateTime 配置选项

由于 SQLite 没有专门存储日期时间的类型，Drift 为 `DateTime` 对象提供两种存储方式：

1. **Unix 时间戳**（默认）
   `dateTime()` 列在数据库中为 INTEGER 类型，存储秒级时间戳，**不包含时区、毫秒精度**。

2. **ISO-8601 字符串**（推荐）
   将 `dateTime()` 列存储为文本类型，精度更高、支持时区，**推荐新项目使用**。

Drift 为了向后兼容，默认使用 Unix 时间戳。
如需启用 ISO 字符串存储，在 `build.yaml` 中修改 `store_date_time_values_as_text` 配置：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          store_date_time_values_as_text: false # 默认值
          # 启用 ISO 8601 字符串
          # store_date_time_values_as_text: true
```

更多日期存储方式、切换方案详见 **DateTime 迁移指南**。

---

注释：
SQL 类型仅作用于数据库。JSON 序列化不受 SQL 类型影响。
例如，布尔值在数据库中存储为 1/0，但在 JSON 中仍序列化为 true/false。
