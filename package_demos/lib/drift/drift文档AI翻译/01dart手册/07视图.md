# 视图

如何将 SQL 视图定义为 Dart 类

你也可以将 SQL 视图定义为 Dart 类。实现方式为：编写一个继承自 `View` 的抽象类。
本示例基于前文的数据库结构，声明了一个视图，用于读取每个分类下的待办事项数量：

```dart
abstract class CategoryTodoCount extends View {
  // Getter 定义了视图所依赖的数据表
  Todos get todos;
  Categories get categories;

  // 自定义表达式可以通过 Getter 命名
  Expression<int> get itemCount => todos.id.count();

  @override
  Query as() =>
      // 视图既可以选择类中定义的表达式列，也可以引用其他表的列
      select([categories.description, itemCount]).from(categories).join([
        innerJoin(todos, todos.category.equalsExp(categories.id)),
      ]);
}
```

---

在 Dart 视图中，遵循以下规则定义结构：

1.  使用**抽象 Getter** 声明视图依赖的源数据表（例如 `TodosTable get todos`）。
2.  使用 `Expression` 类型的 Getter 定义自定义列（例如 `itemCount => todos.id.count()`）。
3.  重写 `as` 方法，定义支撑视图的查询语句。`select` 中引用的列分为两类：
    - 视图自身定义的列（如上例中的 `itemCount`）；
    - 引用自其他数据表的列（如上例中的 `categories.description`）。对于这类引用，**原数据表列中使用的类型转换器等高级 Drift 功能，会自动同步应用到视图列上**。

以上两类列被选中后，都会自动生成到视图对应的数据类中。

最后，需要将视图添加到数据库或访问器的 `views` 参数中：

```dart
@DriftDatabase(tables: [Todos, Categories], views: [CategoryTodoCount])
class MyDatabase extends _$MyDatabase {
```

## 视图中列的可空性

对于 Dart 定义的视图，**通过 `Expression` Getter 定义的表达式列始终为可空类型**。
该行为与 `TypedResult.read` 方法完全一致（该方法用于读取包含自定义列的复杂查询结果）。

引用其他数据表的列，满足以下任一条件则为**可空列**：

- 被引用的列本身是可空的；
- 被引用的表**不来自内连接**（这种情况下整行数据都可能为 `null`）。

以上文的视图为例：

1.  `itemCount` 列为可空类型，因为它是由复杂表达式定义的列；
2.  `description` 列引用自 `categories.description`，为**非空类型**。因为它指向的是视图查询语句的**主表** `categories`。
