# 写入操作（更新、插入、删除）

修改数据库中的数据

---

## 更新与删除

你可以使用 Drift 自动生成的类来更新任意数据行的单个字段：

```dart
Future moveImportantTasksIntoCategory(Category target) {
  // 对于更新操作，我们使用生成类的「伴随类（Companion）」版本。
  // 它会将字段包装在 Value 类型中，可通过 Value.absent() 标记为不更新。
  // 这让我们能够区分「将字段设为 NULL」（category: Value(null)）
  // 和「完全不更新该字段」（category: Value.absent()）。
  return (update(todos)
      ..where((t) => t.title.like('%Important%'))
    ).write(TodosCompanion(
      category: Value(target.id),
    ),
  );
}

Future updateTodo(Todo entry) {
  // 使用 replace 会更新该实体中**非主键**的所有字段。
  // 它会自动匹配相同主键的数据行进行更新。
  // 在本例中，与 entry 拥有相同 id 的行，会被更新为该实体的标题、内容和分类。
  // 由于 where 条件自动生成，该方法**不能**与自定义 where 共用。
  return update(todos).replace(entry);
}

Future feelingLazy() {
  // 删除最旧的 9 条任务
  return (delete(todos)..where((t) => t.id.isSmallerThanValue(10))).go();
}
```

⚠️ **重要警告**：如果在更新或删除操作中**没有显式添加 where 筛选条件**，该语句会影响表中的**所有数据行**！

---

### 数据实体、伴随类 —— 为什么需要区分使用？

你可能注意到，第一个更新操作使用了 `TodosCompanion`，而非直接传入 `Todo` 对象。
Drift 为每张表生成的 `Todo` 类（数据实体类）用于承载**完整的一行数据**；而**部分字段更新**时，推荐使用伴随类。

在上文示例中，我们仅需要更新 `category` 字段，因此使用伴随类。
为什么这是必要的？
如果一个字段被设为 `null`，我们无法判断是需要将数据库列置空，还是保持原值不变。
伴随类中的字段拥有特殊的 `Value.absent()` 状态，明确区分这两种行为。

伴随类还为**插入操作**提供了专用构造函数：所有**非空、无默认值**的字段都会被标记为 `@required`，让你清晰知道插入时必须填写哪些字段。

## 使用 SQL 表达式更新

在某些场景下，你需要基于数据行的**当前值**批量更新多行数据。
一种方案是：先将数据查询到 Dart 对象中，基于结果创建伴随类，再执行更新。
如果更新逻辑可以用 SQL 表达，更高效的方式是使用 `Companion.custom`：

```dart
await db
    .update(db.users)
    .write(UsersCompanion.custom(username: db.users.username.lower()));
```

本例中，用户表的所有用户名将被批量转为**小写**。
由于字段的 `.lower()` 方法在数据库层执行，执行过程中无需将数据加载到 Dart 中，性能更高。

## 插入

你可以轻松向数据表插入任意合法对象。
由于部分字段可以省略（如默认值、自增ID），我们**依旧使用伴随类**。

```dart
// 返回自动生成的主键 ID
Future<int> addTodo(TodosCompanion entry) {
  return into(todos).insert(entry);
}
```

所有自动生成的数据行类都提供构造函数，用于创建对象：

```dart
addTodo(
  TodosCompanion(
    title: Value('Important task'),
    content: Value('Refactor persistence code'),
  ),
);
```

### 规则

- 如果列是**可空**的，或拥有**默认值**（包含自增主键），该字段可以省略
- 其他字段**必须赋值且非空**，否则 `insert` 方法会抛出异常

---

### 批量插入

通过**批量操作**可以高效执行多条插入语句，在 `batch` 中使用 `insertAll` 方法：

```dart
Future<void> insertMultipleEntries() async{
  await batch((batch) {
    // 批量操作内的方法无需单独 await
    // 仅需 await 整个批量操作即可
    batch.insertAll(todos, [
      TodosCompanion.insert(
        title: 'First entry',
        content: 'My content',
      ),
      TodosCompanion.insert(
        title: 'Another entry',
        content: 'More content',
        // 非必填字段仍需用 Value 包装
        category: Value(3),
      ),
      // ...
    ]);
  });
}
```

批量操作与事务类似，所有更新操作**原子性执行**；同时还能优化性能，避免重复编译相同 SQL 语句，非常适合**大批量插入/更新**场景。

## 插入或更新（Upserts）

插入或更新是 sqlite3 新版本提供的特性：**当插入的数据存在主键/唯一约束冲突时，自动执行更新操作**。

该特性允许我们通过主键判断：创建新行，或覆盖已有行。

示例：用户表（邮箱为主键）

```dart
class Users extends Table {
  TextColumn get email => text()();
  TextColumn get name => text()();

  @override
  Set<Column> get primaryKey => {email};
}

Future<int> createOrUpdateUser(User user) {
  return into(users).insertOnConflictUpdate(user);
}
```

调用 `createOrUpdateUser()` 时：

- 如果邮箱已存在 → 更新用户名
- 如果邮箱不存在 → 插入新用户

### 高级自定义插入或更新

以单词统计表为例，记录单词出现次数：

```dart
class Words extends Table {
  TextColumn get word => text()();
  IntColumn get usages => integer().withDefault(const Constant(1))();

  @override
  Set<Column> get primaryKey => {word};
}
```

通过自定义冲突处理，实现：**新单词插入，已有单词计数+1**：

```dart
Future<void> trackWord(String word) {
  return into(words).insert(
    WordsCompanion.insert(word: word),
    onConflict: DoUpdate(
      (old) => WordsCompanion.custom(usages: old.usages + Constant(1)),
    ),
  );
}
```

---

### 唯一约束与冲突目标

`insertOnConflictUpdate` 和 `onConflict: DoUpdate` 底层使用 SQL 的 `DO UPDATE` 语法。
这需要指定**冲突目标**（一组用于检查唯一性的列）。

**默认规则**：Drift 自动使用表的**主键**作为冲突目标。
如果你的表使用**自定义唯一约束**，必须在 `DoUpdate` 中手动指定 `target` 参数：

```dart
class MatchResults extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get teamA => text()();
  TextColumn get teamB => text()();
  BoolColumn get teamAWon => boolean()();

  // 自定义组合唯一约束：两支球队的比赛唯一
  @override
  List<Set<Column<Object>>>? get uniqueKeys => [
    {teamA, teamB},
  ];
}

Future<void> insertMatch(String teamA, String teamB, bool teamAWon) {
  final data = MatchResultsCompanion.insert(
    teamA: teamA,
    teamB: teamB,
    teamAWon: teamAWon,
  );

  return into(matches).insert(
    data,
    onConflict: DoUpdate(
      (old) => data,
      // 手动指定自定义唯一约束作为冲突目标
      target: [matches.teamA, matches.teamB],
    ),
  );
}
```

### 补充说明

1. 该特性需要 **sqlite 3.24.0+** 版本；旧版安卓设备使用 `drift_sqflite` 可能不支持，推荐使用 `NativeDatabases`
2. 执行插入或更新时，返回的 `rowid` 可能不准确

## 插入并返回完整行

使用 `insertReturning` 方法，可以**插入数据后立即返回完整的数据行**（包含所有默认值、自增ID、数据库生成的字段）。

以入门示例的表为例：

```dart
final row = await into(todos).insertReturning(TodosCompanion.insert(
  title: 'A todo entry',
  content: 'A description',
));
```

返回的 `row` 对象已包含正确的自增ID；
如果表包含其他默认值（如 `CURRENT_TIME` 动态时间），也会完整包含在返回结果中。
