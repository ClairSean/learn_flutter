# 安装配置

涵盖将 Drift 集成到项目中的全部必要知识。

Drift 是一款适用于 Dart 和 Flutter 应用的强大数据库库。为了支持**类型安全的 SQL 查询**、数据库校验和数据迁移等高级特性，它使用了编译期运行的构建器和命令行工具。

这意味着，Drift 的集成步骤不只是在 `pubspec` 中添加一个依赖那么简单。本文档将讲解如何在项目中接入 Drift，并为后续学习提供指引。
如果你在集成 Drift 时遇到问题，或对项目有疑问、建议，欢迎在 GitHub 发起讨论与社区分享。
如果你想参考示例应用获取灵感，Drift 仓库中包含了一个跨平台 Flutter 示例项目。

---

## 依赖配置

首先，我们将 Drift 添加到项目的 `pubspec.yaml` 文件中。
除了 Drift 核心依赖（`drift` 运行时库和 `drift_dev` 代码生成器），我们还需要添加对应平台的数据库打开工具包。

```yaml
dependencies:
  drift: ^2.32.1
  drift_flutter: ^0.3.1-wip
  path_provider: ^2.1.5

dev_dependencies:
  drift_dev: ^2.32.1
  build_runner: ^2.13.1
```

你也可以通过以下命令快速完成依赖添加：

```bash
dart pub add drift drift_flutter path_provider dev:drift_dev dev:build_runner
```

## 数据库类

所有使用 Drift 的项目，都至少需要一个**数据库访问类**。
这个类会声明你需要使用的所有数据表，是 Drift 代码生成器的核心入口。

在本示例中，我们假设数据库类定义在 `lib/database.dart` 文件中（你可以将其放在任意 Dart 文件内）。

为了让数据库具备实际功能，我们先添加一个简单的数据表：`TodoItems`，用于存储待办事项，可作为待办清单应用的持久化层。
关于在 Dart 中定义数据表的完整说明，可查看 [Dart 数据表] 页面；如果你更喜欢用 SQL 定义表结构，Drift 也完全支持！相关说明见[此处]。

现在，将以下代码写入 `database.dart`：

```dart
import 'package:drift/drift.dart';

part 'database.g.dart';

class TodoItems extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get title => text().withLength(min: 6, max: 32)();
  TextColumn get content => text().named('body')();
  DateTimeColumn get createdAt => dateTime().nullable()();
}

@DriftDatabase(tables: [TodoItems])
class AppDatabase extends _$AppDatabase {
}
```

此时你会在 `part` 语句和 `extends _$AppDatabase` 处看到分析器警告，**这是正常现象**，因为 Drift 的代码生成器尚未运行。
你可以通过执行以下命令生成代码：

- `dart run build_runner build`：一次性生成所有必需代码
- `dart run build_runner watch`：监听源文件变动，增量生成代码（适合开发阶段使用）

命令执行完成后，包含生成类 `_$AppDatabase` 的 `database.g.dart` 文件会自动创建。
此时你会看到「缺少重写方法」「缺少构造函数」的报错，我们需要完善数据库类：

- 构造函数：告知 Drift 如何打开数据库
- `schemaVersion`：数据库版本号，用于后续数据迁移（初始设为 1 即可）

完善后的 `database.dart`：

```dart
import 'package:drift/drift.dart';
import 'package:drift_flutter/drift_flutter.dart';
import 'package:path_provider/path_provider.dart';

part 'database.g.dart';

class TodoItems extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get title => text().withLength(min: 6, max: 32)();
  TextColumn get content => text().named('body')();
  DateTimeColumn get createdAt => dateTime().nullable()();
}

@DriftDatabase(tables: [TodoItems])
class AppDatabase extends _$AppDatabase {
  // 代码生成后，此类需要定义 schemaVersion 和数据库构造函数
  AppDatabase([QueryExecutor? executor]) : super(executor ?? _openConnection());

  @override
  int get schemaVersion => 1;

  static QueryExecutor _openConnection() {
    return driftDatabase(
      name: 'my_database',
      native: const DriftNativeOptions(
        // 默认情况下，drift_flutter 的 driftDatabase 会将数据库文件
        // 存储在 getApplicationSupportDirectory() 目录
        databaseDirectory: getApplicationSupportDirectory,
      ),
      // 如需 Web 支持，请参考官方文档：https://drift.simonbinder.eu/platforms/web/
    );
  }
}
```

如需自定义数据库打开方式，也可以手动配置连接（手动数据库配置）。

## 后续步骤

恭喜！集成完成后，你的项目已经可以正常使用 Drift 了。
以下代码片段展示了如何打开数据库、执行插入和查询操作：

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  final database = AppDatabase();

  // 插入一条待办数据
  await database
      .into(database.todoItems)
      .insert(
        TodoItemsCompanion.insert(
          title: 'todo: 完成 Drift 集成',
          content: '现在可以编写查询并自定义数据表了。',
        ),
      );
  // 查询所有数据
  List<TodoItem> allItems = await database.select(database.todoItems).get();

  print('数据库中的数据：$allItems');
}
```

Drift 的能力远不止于此！以下文档对你入门 Drift 非常有帮助：

- **Dart 数据表**：讲解如何在 Dart 中自定义数据表；Drift 为表生成的类说明可查看[行数据类]
- **Manager API**：推荐新手或不熟悉 SQL 的开发者使用，语法与主流 ORM 一致，轻松编写大部分查询
- **编写查询**：Drift 生成的类支持编写所有常用 SQL 语句（查询、插入、更新、删除）
- **数据迁移**：后续修改数据库（新增列/表）时，需要编写迁移脚本，Drift 提供完善的迁移工具
- **常见问题**：解答 Drift 项目中最常见的疑问

熟悉基础用法后，可查看官方功能概览，了解更多高级特性：事务、自动化迁移工具、多平台支持等。
