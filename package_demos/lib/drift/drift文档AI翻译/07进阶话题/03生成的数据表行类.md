# 生成的数据表行类

本文概述 Drift 生成的、用于读写数据库数据表的各类类。

对于你定义的每一张数据表，Drift 都会生成两个关联类：

1. **行类**：该类代表数据表中的一整行数据。Drift 在执行数据表查询时，会自动返回该类的实例，让你以类型安全的方式访问行数据。
2. **伴随类（Companion）**：行类代表数据库中完整的一行数据，但开发中有时需要**部分行数据**（例如插入数据时自增主键还未生成、仅更新部分字段）。针对这种场景，Drift 生成伴随类，主要用于数据插入与更新操作。

Drift 生成的行类内置了相等判断、哈希值计算、基础序列化支持，同时自带 `copyWith` 方法，方便快速修改数据。

---

## 示例

下面用一个存储用户名的简单表，演示生成的行类与伴随类的作用：

```dart
class Users extends Table {
  late final id = integer().autoIncrement()();
  late final username = text()();
}
```

针对这张表，Drift 会生成一个 `User` 类（省略部分便捷工具方法，简化版如下）：

```dart
// Drift 生成的行类简化版本
class User {
  final int id;
  final String username;

  const User({required this.id, required this.username});

  @override
  String toString() {
    // ...
  }

  @override
  int get hashCode => Object.hash(id, username);

  @override
  bool operator ==(Object other) =>
      identical(this, other) ||
      (other is User && other.id == this.id && other.username == this.username);
}
```

可以看到 `User.id` 是不可空字段，对应数据库中该字段同样不可空。
但插入新用户时，主键 `id` 由数据库自动生成，你无法手动指定。因此 Drift 额外提供伴随类来表示**部分行数据**：

```dart
class UsersCompanion extends UpdateCompanion<User> {
  final Value<int> id;
  final Value<String> username;

  const UsersCompanion({
    this.id = const Value.absent(),
    this.username = const Value.absent(),
  });

  UsersCompanion.insert({
    this.id = const Value.absent(),
    required String username,
  }) : username = Value(username);

  @override
  Map<String, Expression> toColumns(bool nullToAbsent) {
    // ...
  }
}
```

伴随类中所有字段都被 `Value` 类包裹，用来标识该字段**是否需要赋值**。

---

## 使用行类

通过这两个生成的类，你可以以类型安全、结构化的方式创建和读取数据库行：

```dart
await db.managers.users.create((row) => row(username: 'firstuser'));
```

注意：管理器（Manager）API 不使用伴随类，创建新行的方法会把必填字段作为必填参数传入。

## 数据类命名规则

默认情况下，数据类名称由数据表名称自动推导：

1. 表名以 `s` 结尾：去掉末尾 `s` 作为类名
   示例：`Users` → `User`
2. 表名不以 `s` 结尾：表名后追加 `Data`
   示例：`UserInfo` → `UserInfoData`

想要自定义生成类名，可以使用 `@DataClassName` 注解：

```dart
@DataClassName('Category')
class Categories extends Table {
  late final id = integer().autoIncrement()();
  late final title = text()();
}

void readCategories(Database db) async {
  // 通过 @DataClassName，生成类名为 Category，而非 Categorie
  final List<Category> categories = await db.categories.all().get();
  print('当前分类：$categories');
}
```

## 伴随类与 Value 类型

伴随类用于表示**不完整的部分行数据**，设计它主要有两个原因：

1. Dart 是空安全类型系统：如果只有一种行类，数据库自动生成的值（如自增主键）在插入时必须定义为可空，但可空类型又不适合查询场景。
2. 需要区分 **SQL NULL（空值）** 和 **未设置字段**：更新时，把字段设为 NULL ≠ 不修改该字段（保留原值）；但 Dart 只有一个 `null`，因此需要单独结构区分。

为解决该问题，伴随类使用 Drift 自带的 `Value` 类表示部分行：`Value` 可以存储值（可空），也可以显式标记**字段不存在/不赋值**。

```dart
// 插入数据
await db.users.insertOne(
  UsersCompanion(
    id: Value.absent(), // 主键自增，不手动赋值
    username: Value('user'), // 指定用户名
  ),
);

// 更新数据：仅修改用户名，id 保持不变
await (db.update(db.users)..where((tbl) => tbl.id.equals(1))).write(
  UsersCompanion(username: Value("更新后的名字")),
);
```

1. `id` 是自增主键，数据库自动生成，使用 `Value.absent()`；同时这也是默认值，可以省略不写。
2. 为简化常用插入场景，伴随类提供 `.insert()` 构造器，不需要的字段可以不用包裹 `Value`，上面插入代码可简写为：
   `UsersCompanion.insert(username: 'user')`

### 使用 SQL 表达式更新

伴随类还提供 `.custom` 方法，用于更新/删除时**混合普通值与 SQL 表达式**。
例如把用户表所有用户名转为小写：

```dart
await db
    .update(db.users)
    .write(UsersCompanion.custom(username: db.users.username.lower()));
```

## 自定义数据类

Drift 默认生成的数据类能满足大多数场景，但你也可以**自定义类来表示数据表行**。
适合场景：自定义类需要继承、实现、混入其他类，或是想搭配 `json_serializable` 等其他代码生成工具。

> 文档说明：行类、数据类是同一个概念，都代表数据表一行数据。

### @UseRowClass 自定义行类

给数据表加上 `@UseRowClass` 注解即可使用自定义行类：

```dart
@UseRowClass(User)
class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text()();
  DateTimeColumn get birthday => dateTime()();
}

class User {
  final int id;
  final String name;
  final DateTime birthday;

  User({required this.id, required this.name, required this.birthday});
}
```

默认要求：

1. 自定义类必须有无命名构造器；
2. 构造器参数名必须和数据表字段 getter 名一致；
3. 参数类型必须和字段类型完全一致（包括可空性、类型转换器）。

额外说明：

- 自定义类可以有额外字段/构造参数，只要是非必填即可，Drift 映射数据时会忽略；
- 数据表可以有自定义类中不存在的字段，Drift 读取时不会加载这些字段。

### 使用 Dart 记录类型（Record）

想要极简的行数据结构，可以让 Drift 生成匹配的 Dart 记录类型：

```dart
@UseRowClass(Record) // Drift 自动使用 ({int id, String name}) 记录
class Categories extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text()();
}
```

也可以显式指定记录类型：

```dart
typedef TodoItem = ({int id, String content, int author});

@UseRowClass(TodoItem)
class TodoItems extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get content => text()();
  IntColumn get author => integer().references(Users, #id)();
}
```

### 使用自定义命名构造器

默认 Drift 使用无命名构造器映射行数据，你可以在 `@UseRowClass` 中指定构造器名：

```dart
@UseRowClass(User, constructor: 'fromDb')
class Users extends Table {
  // ...
}

class User {
  final int id;
  final String name;
  final DateTime birthday;

  User.fromDb({required this.id, required this.name, required this.birthday});
}
```

### 静态与异步工厂构造器

Drift 2.0+ 支持在注解中指定**静态方法/异步静态方法**作为行数据加载入口。方法返回行类本身或 `Future<行类>`，适合 SQL 转 Dart 需要异步处理的场景：

```dart
class User {
  // ...

  static Future<User> load(int id, String name, DateTime birthday) async {
    // 异步逻辑
  }
}
```

### 自定义伴随类

大多数场景下默认伴随类足够用于插入更新。如果你想用自定义行类执行插入，只需让它实现 `Insertable<T>` 接口（T 为当前行类本身）：

```dart
class User implements Insertable<User> {
  final int id;
  final String name;
  final DateTime birthDate;

  User({required this.id, required this.name, required this.birthDate});

  @override
  Map<String, Expression> toColumns(bool nullToAbsent) {
    return UsersCompanion(
      id: Value(id),
      name: Value(name),
      birthDate: Value(birthDate),
    ).toColumns(nullToAbsent);
  }
}
```

手动实现接口比较繁琐，Drift 提供 `write_to_columns_mixins` 构建选项：启用后会自动生成混入类，直接混入即可实现 `toColumns`：

```dart
class User with UsersToColumns {
  // ...
}
```

## 在 Drift SQL 文件中使用自定义行类

在 `.drift` SQL 文件定义表时，在表末尾使用 `WITH` 关键字绑定现有 Dart 行类；同时记得在 drift 文件导入对应的 Dart 文件。

```sql
import 'user.dart';

CREATE TABLE users(
  id INTEGER NOT NULL PRIMARY KEY,
  name TEXT NOT NULL,
  birth_date DATETIME NOT NULL
) WITH User;
```

视图同样支持该语法，在视图名后追加 `WITH 类名`：

```sql
CREATE VIEW my_view WITH ExistingClass AS SELECT ...
```

也可以指定自定义命名构造器：

```sql
CREATE TABLE users(
  id INTEGER NOT NULL PRIMARY KEY,
  name TEXT NOT NULL,
  birth_date DATETIME NOT NULL
) WITH User.myNamedConstructor;
```

### 查询语句绑定自定义行类

Drift 文件中的命名查询也支持绑定现有行类，在查询名后加 `WITH` 即可：

```sql
import 'my_existing_class.dart';

/*
对应 Dart 类：
class MyExistingClass {
  final String name;
  final double avgAge;
  MyExistingClass(this.name, this.avgAge);
}
*/

myQuery WITH MyExistingClass: SELECT name, AVG(age) AS avg_age FROM entries GROUP BY category;
```

也可指定命名构造器：

```sql
/*
class MyExistingClass {
  final String name;
  final double avgAge;
  MyExistingClass.fromSql(this.name, this.avgAge);
}
*/

myQuery WITH MyExistingClass.fromSql: SELECT name, AVG(age) AS avg_age FROM entries GROUP BY category;
```

### 查询结果匹配规则

即使不绑定自定义类，Drift 也会对不同查询生成不同结果结构，这些规则同时约束自定义类的字段定义：

1. **嵌套表**：查询使用 `表.**` 语法时，Drift 会把整表字段打包为嵌套对象，而非逐个生成字段；
2. **嵌套列表结果**：`LIST()` 宏可以把子查询结果转为列表；
3. **单表全量查询**：`SELECT * FROM 表` 会直接使用该表自带的数据类，不生成新类；
4. **单列查询**：只查一个字段时直接返回值本身，不生成结果类；
5. **普通结果集**：不属于以上情况的所有查询。

对应自定义类字段要求：

1. 嵌套表 `表.**`：字段类型必须匹配该表行类（默认生成/自定义均可）；
2. `LIST()` 列表结果：字段必须是 `List<T>`，T 匹配子查询结果类型；
3. 单表结果：直接使用数据表行类即可；
4. 单列结果：直接使用对应类型，或单层包装类；
5. 普通结果集：类字段名必须和查询列名一致，类型可赋值兼容（Drift 会自动适配类型转换器）。

示例：

```sql
CREATE TABLE employees(
  id INTEGER NOT NULL PRIMARY KEY,
  name TEXT NOT NULL,
  supervisor INTEGER REFERENCES employees(id)
);

employeeWithStaff WITH EmployeeWithStaff: SELECT
    self.** ,
    supervisor.name,
    LIST(SELECT * FROM employees WHERE supervisor = self.id) AS staff
  FROM employees AS self
    INNER JOIN employees supervisor ON supervisor.id = self.supervisor
  WHERE id = ?;
```

对应自定义类：

```dart
class IdAndName {
  final int id;
  final String name;
  IdAndName(this.id, this.name);
}

class EmployeeWithStaff {
  final Employee self;
  final String supervisor;
  final List<IdAndName> staff;

  EmployeeWithStaff(this.self, this.supervisor, this.staff);
}
```

# JSON 序列化

Drift 生成的行类支持 JSON 互相转换。

> Drift 官方说明：
> 内置序列化是早期版本加入的功能，设计简陋，仅能满足简单序列化需求。
> `json_serializable`、`built_value`、`freezed` 等专业序列化库功能更强，**官方不推荐使用内置序列化**。
> 内置序列化违反关注点分离原则，后续不会再扩展功能。最佳实践：自定义行类 + 专业序列化库。

基础用法：

```dart
final User user = User.fromJson({'id': 3, 'username': 'awesomeuser'});
print('反序列化用户：${user.username}');
```

## JSON 键名规则

1. Dart 类定义的数据表：JSON 键名使用字段 getter 名称

```dart
class Todos extends Table {
  late final id = integer().autoIncrement()();
  late final title = text()();
  late final createdAt = dateTime().withDefault(currentDateAndTime)();
}
```

JSON：

```json
{
  "id": 1,
  "title": "Todo 1",
  "createdAt": "2024-02-29T12:00:00Z"
}
```

2. Drift SQL 文件定义的表/视图：JSON 键名使用 SQL 原始列名

```sql
-- JSON 键：id, title, created_at
CREATE VIEW todos_view AS SELECT * FROM todos;
```

## 自定义 JSON 键名

使用 `@JsonKey` 注解自定义 JSON 键名。
注意：Drift 自带的 `@JsonKey` 和 `json_annotation` 的 `@JsonKey` **不兼容、不是同一个类**。

示例：

```dart
@JsonKey('created')
late final createdAt = dateTime().withDefault(currentDateAndTime)();
```

JSON：

```json
{
  "id": 1,
  "title": "Todo 1",
  "created": "2024-02-29T12:00:00Z"
}
```

想要统一使用 SQL 原始列名作为 JSON 键，在 `build.yaml` 开启配置：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          use_sql_column_name_as_json_key: true
```

更多 SQL 自定义列名规则，可参考官方列名文档。
