# 类型转换器

通过类型转换器在数据表列中存储更复杂的数据

Drift 开箱即支持多种基础数据类型，但有时你需要存储更复杂的自定义数据。通过**类型转换器（TypeConverters）**即可实现这一需求。
在本示例中，我们将使用 `json_serializable` 包将自定义对象存储到文本列中。Drift 支持为**任意 Dart 类型**编写类型转换器，这里使用该包只是为了简化示例。

---

## 在 Dart 中使用类型转换器

```dart
import 'dart:convert';

import 'package:drift/drift.dart';
import 'package:json_annotation/json_annotation.dart' as j;

part 'converters.g.dart';

@j.JsonSerializable()
class Preferences {
  bool receiveEmails;
  String selectedTheme;

  Preferences(this.receiveEmails, this.selectedTheme);

  factory Preferences.fromJson(Map<String, dynamic> json) =>
      _$PreferencesFromJson(json);

  Map<String, dynamic> toJson() => _$PreferencesToJson(this);
}
```

接下来，我们需要告诉 Drift 如何将 `Preferences` 对象存入数据库，为此编写一个类型转换器：

```dart
// 将偏好配置对象存储为字符串
class PreferenceConverter extends TypeConverter<Preferences, String>
    with JsonTypeConverter2<Preferences, String, Map<String, Object?>> {
  const PreferenceConverter();

  @override
  Preferences fromSql(String fromDb) {
    return Preferences.fromJson(json.decode(fromDb) as Map<String, dynamic>);
  }

  @override
  String toSql(Preferences value) {
    return json.encode(value.toJson());
  }

  @override
  Preferences fromJson(Map<String, Object?> json) {
    return Preferences.fromJson(json);
  }

  @override
  Map<String, Object?> toJson(Preferences value) {
    return value.toJson();
  }
}
```

> 说明：本示例使用了 `JsonTypeConverter2`，旧版转换器已废弃，将在 Drift 下一个大版本中移除。

最后，在数据表定义中使用这个转换器：

```dart
class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text()();

  TextColumn get preferences =>
      text().map(const PreferenceConverter()).nullable()();
}
```

生成的 `User` 数据类会拥有一个**`Preferences` 类型**的 `preferences` 字段。
Drift 会在查询、更新、插入语句中自动处理对象的存储与读取，该特性同样支持自定义编译查询。

### 类型转换器常见问题

1. 如果你的转换器返回的对象**不支持值比较**，生成的数据类也无法进行值比较。建议为这类类重写 `==` 运算符和 `hashCode`。
2. Drift 默认生成部分文件，暂不支持自动导入依赖。如果数据表和数据库类分属不同文件，需要手动为数据表文件添加类型转换器的导入语句。**模块化代码生成**也可以解决这个问题。

## JSON 转换快捷工具

由于 JSON 类型转换非常常用，Drift 提供了内置助手方法。我们可以直接在 `Preferences` 类中声明转换器：

```dart
static JsonTypeConverter2<Preferences, String, Object?> converter =
    TypeConverter.json2(
      fromJson: (json) => Preferences.fromJson(json as Map<String, Object?>),
      toJson: (pref) => pref.toJson(),
    );
```

### 为什么使用 JSON 转换器？

默认情况下，类型转换器仅作用于 **Dart ↔ SQL** 的映射。Drift 会为数据类生成 `fromJson`/`toJson` 方法，默认不会应用类型转换器。
通过混入 `JsonTypeConverter`，你可以让转换器同时生效于 JSON 序列化。

---

## 使用 JSONB 格式

从 **SQLite 3.45.0** 版本开始，支持原生的 JSONB 二进制格式存储 JSON 数据。官方文档说明：

> JSONB 相比普通文本 JSON 的优势：体积缩小 5%~10%，CPU 处理耗时降低一半以上。SQLite 内置 JSON 函数同时支持文本 JSON 和二进制 JSONB。

从 Drift 2.24.0 开始，内置支持 JSONB。复用上面的 `Preferences` 类，编写 JSONB 转换器：

```dart
static JsonTypeConverter2<Preferences, Uint8List, Object?> binaryConverter =
    TypeConverter.jsonb(
      fromJson: (json) => Preferences.fromJson(json as Map<String, Object?>),
      toJson: (pref) => pref.toJson(),
    );
```

将其应用到**二进制列**：

```dart
class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text()();

  BlobColumn get preferences =>
      blob().map(Preferences.binaryConverter).nullable()();
}
```

### JSONB 使用建议

✅ **适用场景**：在 SQL 语句中直接解析/操作 JSON 数据
❌ **不推荐**：仅在 Dart 层读取 JSON（dart:convert 的原生解析性能更优）
⚠️ **兼容性**：SQLite 的 JSONB 是专属格式，会降低数据库的可移植性

## 隐式枚举转换器

枚举类型是类型转换器的常用场景，Drift 内置了两种枚举列类型，无需手动编写转换器：

### 1. 整数枚举（存储枚举索引）

```dart
enum Status {
  none,
  running,
  stopped,
  paused
}

class Tasks extends Table {
  IntColumn get id => integer().autoIncrement()();
  IntColumn get status => intEnum<Status>()();
}
```

### 2. 文本枚举（存储枚举名称）

```dart
class Tasks extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get status => textEnum<Status>()();
}
```

### ⚠️ 枚举使用警告

- **`intEnum`**：**绝对不要在枚举中间插入新值**，会导致索引错乱，已有数据解析错误。新增值请放在枚举末尾。
- **`textEnum`**：**不要重命名枚举值**，否则数据库中存储的字符串无法匹配，会抛出异常。
- 枚举列**不支持叠加其他类型转换器**。

## 在 Drift 文件中使用类型转换器

类型转换器也可以在 `.drift` 文件中使用。假设 `Preferences` 和 `PreferenceConverter` 定义在 `preferences.dart` 中，先导入该文件：

```sql
import 'preferences.dart';

CREATE TABLE users (
  id INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT,
  name TEXT,
  preferences TEXT MAPPED BY `const PreferenceConverter()`
);
```

### 推荐配置

开启 `apply_converters_on_variables` 构建选项（默认开启），Drift 会自动为查询变量应用转换器：

```sql
SELECT * FROM users WHERE preferences = ?
```

此时参数会被推断为 `Preferences` 类型，而非 `String`。

### 枚举支持

Drift 文件内置支持隐式枚举转换器：

```sql
import 'status.dart';

CREATE TABLE tasks (
  id INTEGER NOT NULL PRIMARY KEY,
  status ENUM(Status)
);
```

## 类型转换器与 JSON 序列化

默认情况下，类型转换器**仅作用于数据库读写**，不影响 JSON 序列化。
如果需要让转换器同时适配 JSON，让转换器**混入 `JsonTypeConverter2`** 即可。

### v2 版与旧版转换器区别

- 旧版 `JsonTypeConverter`：强制将 JSON 序列化为字符串，容易导致**双重 JSON 编码**问题；
- 新版 `JsonTypeConverter2`：支持独立的 SQL/JSON 类型，完美解决嵌套序列化问题。

---

## 类型转换器与 SQL 查询

在读写数据行时，Drift 会自动应用类型转换器；但在**自定义 SQL 表达式**中，不会自动应用。

### 正确比较值

- ❌ `equals`：直接比较数据库原始值，**不应用转换器**
- ✅ `equalsValue`：自动应用转换器后再比较

### 手动转换

如果需要手动转换，使用：

```dart
column.converter.toSql(对象); // Dart → SQL
column.converter.fromSql(原始值); // SQL → Dart
```

---

### 核心总结

1. **基础用法**：继承 `TypeConverter`，实现 `fromSql`/`toSql`，用 `map()` 绑定到列；
2. **JSON 快捷方式**：使用 `TypeConverter.json2` 简化 JSON 转换；
3. **枚举**：直接用 `intEnum`/`textEnum`，避免修改枚举结构；
4. **JSONB**：适合 SQL 层操作 JSON，Dart 层优先用普通文本 JSON；
5. **查询**：用 `equalsValue` 替代 `equals`，确保转换器生效。
