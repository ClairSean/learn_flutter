# 测试

为 Drift 数据库编写单元测试的指南

使用 Drift 的 Flutter 应用始终可以通过在真实设备上运行集成测试来进行测试。本指南专注于为基于 Drift 编写的数据库编写**单元测试**。这些测试可以直接在你的电脑上运行和调试，无需额外配置，也不需要物理设备即可执行。

## 配置

在本指南中，我们将测试一个用于存储用户名的极简数据库。与常规 Drift 数据库相比，唯一重要的改动是**构造函数**：我们显式声明 `QueryExecutor` 参数，而非使用无参构造函数向父类传递固定的执行器（如 `NativeDatabase`）。

```dart
import 'package:drift/drift.dart';

part 'database.g.dart';

class Users extends Table {
  IntColumn get id => integer().autoIncrement()();
  TextColumn get name => text()();
}

@DriftDatabase(tables: [Users])
class MyDatabase extends _$MyDatabase {
  // 显式接收查询执行器
  MyDatabase(QueryExecutor e) : super(e);

  @override
  int get schemaVersion => 1;

  /// 创建用户并返回其ID
  Future<int> createUser(String name) {
    return into(users).insert(UsersCompanion.insert(name: name));
  }

  /// 将ID为[id]的用户名修改为[newName]
  Future<void> updateName(int id, String newName) {
    return update(users).replace(User(id: id, name: newName));
  }

  /// 监听ID为[id]的用户数据变化
  Stream<User> watchUserWithId(int id) {
    return (select(users)..where((u) => u.id.equals(id))).watchSingle();
  }
}
```

## 编写测试

我们可以使用 `NativeDatabase.memory()` 创建一个**内存版数据库**，替代 `FlutterQueryExecutor` 或其他实现。
使用 `test` 包中的 `setUp` 和 `tearDown` 方法来初始化和关闭数据库是最佳实践：

```dart
import 'package:drift/drift.dart';
import 'package:drift/native.dart';
import 'package:test/test.dart';
// 导入上方定义的数据库文件（可用于测试任意Drift数据库）
import 'database.dart';

void main() {
  late MyDatabase database;

  setUp(() {
    database = MyDatabase(
      DatabaseConnection(
        // 内存数据库，测试专用
        NativeDatabase.memory(),
        // 组件测试推荐开启，避免测试报错
        closeStreamsSynchronously: true,
      ),
    );
  });

  // 测试结束后关闭数据库
  tearDown(() async {
    await database.close();
  });
}
```

### 同步关闭流

默认情况下，取消订阅 Drift 创建的查询流时，流会保持开启一个事件循环周期。这对 Flutter 应用很有用（例如 `StreamBuilder` 因重建频繁重订阅流）。
但在 **Flutter 组件测试** 中，测试结束后保留定时器会报错。为避免该问题，向数据库传入配置了 `closeStreamsSynchronously: true` 的 `DatabaseConnection`。

配置完成后，我们就可以编写测试用例了：

```dart
test('可以创建用户', () async {
  final id = await database.createUser('测试用户');
  final user = await database.watchUserWithId(id).first;

  expect(user.name, '测试用户');
});

test('修改用户名时流会发出新数据', () async {
  final id = await database.createUser('初始名称');

  // 预期流会依次发出两个用户名
  final expectation = expectLater(
    database.watchUserWithId(id).map((user) => user.name),
    emitsInOrder(['初始名称', '修改后名称']),
  );

  await database.updateName(id, '修改后名称');
  await expectation;
});
```

## 测试中的确定性时间

在 Flutter 组件测试（以及使用 `fakeAsync` 或 `withClock` 的纯 Dart 测试）中，你可能需要**模拟当前日期时间**，让测试结果具备确定性，或稳定复现依赖时间的业务逻辑。

在 Dart 中，我们通常使用 `clock.now()` 而非 `DateTime.now()` 来监听模拟时钟。
SQLite 是 C 语言编写的库，无法感知 Dart 中的模拟时钟，这意味着 `CURRENT_TIMESTAMP` 或 `datetime('now')` 仍会使用系统真实时间。

通过安装**自定义虚拟文件系统（VFS）** 可以解决这个问题。它会拦截 SQLite 库的所有 IO 操作，让时间完全可控。
对于测试，`sqlite3_test` 包提供的预编译 `TestSqliteFileSystem` 可以监听 `withClock` 设置的模拟时间（包含组件测试）。

文件系统建议在 `setUpAll` 中配置，并在 `tearDownAll` 中注销。
使用该功能的每个测试文件，都需要为 VFS 注册**唯一名称**，因为多个测试可能在同一进程中并行运行。

```dart
import 'package:drift/drift.dart';
import 'package:drift/native.dart';
import 'package:file/local.dart';
import 'package:sqlite3/sqlite3.dart';
import 'package:sqlite3_test/sqlite3_test.dart';
import 'package:uuid/uuid.dart';

void main() {
  late TestSqliteFileSystem vfs;

  setUpAll(() {
    // 创建测试专用VFS，使用唯一名称
    vfs = TestSqliteFileSystem(fs: const LocalFileSystem(), name: const Uuid().v4());
    sqlite3.registerVirtualFileSystem(vfs);
  });

  tearDownAll(() => sqlite3.unregisterVirtualFileSystem(vfs));
}
```

你仍可以按照「编写测试」章节的方式初始化数据库。
在测试中，所有与时间相关的 SQLite 函数都会使用 Dart 中的模拟时间：

```dart
test('测试依赖数据库时间的逻辑', () async {
  // 模拟人类首次登月时间
  final moonLanding = DateTime.utc(1969, 7, 20, 20, 18, 04);

  // 覆盖全局时钟
  await withClock(Clock.fixed(moonLanding), () async {
    // 直接读取时间（触发器、默认值等间接调用也会使用模拟时间）
    final row = await database.selectExpressions([
      currentDateAndTime.year,
    ]).getSingle();

    expect(row.read(currentDateAndTime.year)!, 1969);
  });
});
```
