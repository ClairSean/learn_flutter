# 隔离区

在多个 Dart 隔离区中访问 Drift 数据库

由于 sqlite3 是一个同步的 C 语言库，在主隔离区中访问数据库会引发阻塞式 IO 操作，导致应用响应性下降。为解决这个问题，Drift 可以创建一个长生命周期的隔离区来执行 SQL 语句。按照官方入门指南的推荐写法，使用 `NativeDatabase.createInBackground` 时，Drift 会自动为你管理隔离区，无需额外配置。本文将介绍何时需要高级隔离区配置，以及对应的实现方案。

---

## 何时使用 Drift 隔离区

- Drift 的默认配置**已经使用隔离区**，避免同步 IO 阻塞主隔离区；
- 你可以在不同隔离区中打开两个独立的 Drift 数据库，无需特殊配置或 Drift API。这些数据库甚至可以指向同一个文件，但流查询不会在独立实例间同步；
- 如果你需要**在多个隔离区共享同一个 Drift 数据库**，则必须进行额外配置 —— 这正是 Drift 隔离区 API 的设计目的！

---

## 简介

默认配置适用于大多数应用，但部分场景需要完全控制 Drift 的隔离区管理方式，例如：

- 你希望在 `compute()`、`Isolate.run` 或其他隔离区中使用 Drift；
- 你需要在后台工作器中访问 Drift 数据库；
- 你希望自定义 Drift 创建隔离区的方式，而非使用默认配置。

如果尝试直接跨隔离区传递 Drift 数据库实例，会触发**非法对象传输**异常：

```dart
Future<void> invalidIsolateUsage() async {
  final database = MyDatabase(NativeDatabase.memory());

  // 这种写法无法生效：Drift 数据库包含流、Future 等异步对象，
  // 无法直接跨隔离区序列化传输
  await Isolate.run(() async {
    await database.batch((batch) {
      // ...
    });
  });
}
```

遗憾的是，Drift 无法通过简单修改实现直接传输数据库实例 —— 实现流查询、高级事务 API 需要大量可变状态。不过，通过少量额外配置，你可以使用 Drift API 获取两个**通过隔离区通道同步**的数据库实例：在一个实例上写入的数据，在另一个隔离区中可以读取，甚至会自动更新流查询。简单来说，你可以获得一个**跨隔离区共享的逻辑数据库实例**。

---

## 简易共享方案

从 Drift 2.5 开始，使用 `computeWithDatabase` 可以轻松在临时隔离区中执行耗时任务。该方法会自动创建隔离区，并在新隔离区中打开第二个数据库实例。

由于后台数据库实例需要与前台主实例同步，你的数据库类需要提供一个接收自定义 `QueryExecutor`（Drift 用于表示底层数据库的类）的构造方法：

```dart
@DriftDatabase(tables: [SomeTable] /* ... */)
class MyDatabase extends _$MyDatabase {
  // 该构造方法既可以使用入门指南中的默认连接，也支持自定义连接
  MyDatabase([QueryExecutor? executor]) : super(executor ?? _openConnection());

  @override
  int get schemaVersion => 1;
}

Future<void> insertBulkData(MyDatabase database) async {
  // computeWithDatabase 是 package:drift/isolate.dart 提供的扩展方法
  await database.computeWithDatabase(
    computation: (database) async {
      // 在独立隔离区中执行耗时操作，与主数据库互通数据
      final rows = await _complexAndExpensiveOperationToFetchRows();
      await database.batch((batch) {
        batch.insertAll(database.someTable, rows);
      });
    },
    connect: (connection) {
      // 使用临时连接创建第二个数据库实例
      // 数据库类必须支持传入连接的构造方法
      return MyDatabase(connection);
    },
  );
}
```

如示例所示，`computeWithDatabase` 非常适合执行批量插入等耗时任务。

### 内部原理

1. 建立一对 `SendPort`/`ReceivePort` 端口，用于传输数据库调用指令；
2. 通过 `Isolate.run` 创建新隔离区，基于端口创建原始数据库连接；
3. 新隔离区执行 `connect` 回调，创建与主实例通信的第二个数据库实例；
4. 执行 `computation` 耗时任务；
5. Drift 自动处理连接的关闭与资源释放。

如果你不想让 Drift 自动创建隔离区（例如使用 `compute` 而非 `Isolate.run`），可以通过 `serializableConnection()` API 手动实现：

```dart
Future<void> customIsolateUsage(MyDatabase database) async {
  final connection = await database.serializableConnection();

  await Isolate.run(() async {
    // 数据库对象无法跨隔离区共享，但连接可以
    final databaseForIsolate = MyDatabase(await connection.connect());

    try {
      await databaseForIsolate.batch((batch) {
        // (...)
      });
    } finally {
      databaseForIsolate.close();
    }
  }, debugName: 'My custom database task');
}
```

## 手动管理 Drift 隔离区

除了使用 `NativeDatabase.createInBackground` 或 `computeWithDatabase`，你也可以手动创建**支持跨隔离区共享**的数据库连接。

Drift 提供了 `DriftIsolate` 类，代表一个可在其他隔离区访问的内部数据库服务。通过 `DriftIsolate.spawn()` 即可创建：

```dart
Future<DriftIsolate> createIsolateWithSpawn() async {
  final token = RootIsolateToken.instance!;
  return await DriftIsolate.spawn(() {
    // 该函数运行在新隔离区中，必须先初始化消息器以使用平台通道
    BackgroundIsolateBinaryMessenger.ensureInitialized(token);

    // 必须返回要使用的数据库连接
    return LazyDatabase(() async {
      // 此代码运行在后台隔离区，Flutter 3.7 才开始支持平台通道
      // 旧版本的兼容方案见本文后续章节
      final dbFolder = await getApplicationDocumentsDirectory();
      final path = p.join(dbFolder.path, 'app.db');

      return NativeDatabase(File(path));
    });
  });
}
```

你也可以完全手动创建隔离区：

```dart
Future<DriftIsolate> createIsolateManually() async {
  final receiveIsolate = ReceivePort('receive drift isolate handle');
  await Isolate.spawn<SendPort>((message) async {
    final server = DriftIsolate.inCurrent(() {
      // 返回懒加载数据库或连接
    });

    // 通知原始隔离区服务已创建
    message.send(server);
  }, receiveIsolate.sendPort);

  final server = await receiveIsolate.first as DriftIsolate;
  receiveIsolate.close();
  return server;
}
```

创建 `DriftIsolate` 服务后，在任意隔离区调用 `connect()` 即可连接：

```dart
import 'package:drift/isolate.dart';

@DriftDatabase(tables: [SomeTable] /* ... */)
class MyDatabase extends _$MyDatabase {
  MyDatabase([QueryExecutor? executor]) : super(executor ?? _openConnection());

  @override
  int get schemaVersion => 1;
}

void main() async {
  final isolate = await createIsolate();

  // 连接服务获取连接，创建数据库实例
  // singleClientMode：关闭连接时自动销毁隔离区
  final database = MyDatabase(await isolate.connect(singleClientMode: true));

  // 数据库用法与普通实例完全一致
}
```

如果需要在**非异步环境**中创建数据库（例如依赖注入框架），可以使用 `DatabaseConnection.delayed` 延迟构造：

```dart
MyDatabase(
  DatabaseConnection.delayed(
    Future.sync(() async {
      final isolate = await createIsolate();
      return isolate.connect(singleClientMode: true);
    }),
  ),
);
```

Drift 会在执行第一条查询时自动建立连接。

### 避免与 LazyDatabase 混用

Drift 文档推荐使用 `LazyDatabase` 异步初始化连接，但**不建议**将其用于多客户端的 `DriftIsolate` —— 它仅共享查询执行器，无法同步流查询。`DatabaseConnection.delayed` 可以保证流查询正常同步。

## 旧版 Flutter 兼容方案

Flutter 3.7 之前，后台隔离区不支持平台通道，因此无法直接调用 `getApplicationDocumentsDirectory` 等方法。解决方案：**在主隔离区计算数据库路径，再传递给后台隔离区**。

```dart
import 'package:path/path.dart' as p;
import 'package:path_provider/path_provider.dart';

Future<DriftIsolate> _createDriftIsolate() async {
  // 主隔离区中计算数据库路径
  final dir = await getApplicationDocumentsDirectory();
  final path = p.join(dir.path, 'db.sqlite');
  final receivePort = ReceivePort();

  await Isolate.spawn(
    _startBackground,
    _IsolateStartRequest(receivePort.sendPort, path),
  );

  // 接收后台隔离区返回的 DriftIsolate
  return await receivePort.first as DriftIsolate;
}

void _startBackground(_IsolateStartRequest request) {
  // 后台隔离区入口：创建数据库
  final executor = NativeDatabase(File(request.targetPath));
  // 直接在当前后台隔离区创建服务，避免重复创建隔离区
  final driftIsolate = DriftIsolate.inCurrent(
    () => DatabaseConnection(executor),
  );
  // 将服务回传给主隔离区
  request.sendDriftIsolate.send(driftIsolate);
}

// 封装发送端口和路径，隔离区入口函数仅支持一个参数
class _IsolateStartRequest {
  final SendPort sendDriftIsolate;
  final String targetPath;

  _IsolateStartRequest(this.sendDriftIsolate, this.targetPath);
}
```

同样使用延迟连接创建数据库实例：

```dart
DatabaseConnection createDriftIsolateAndConnect() {
  return DatabaseConnection.delayed(
    Future.sync(() async {
      final isolate = await _createDriftIsolate();
      return await isolate.connect(singleClientMode: true);
    }),
  );
}
```

### 初始化与后台隔离区

Dart 隔离区**不共享内存**，全局变量在主隔离区和后台隔离区中相互独立。
例如：使用 `package:sqlite3` 的 `open.overrideFor` 时，**必须在实际打开数据库的隔离区中调用**（即 `_startBackground` 函数内）。其他全局服务（如 `get_it`）也需要在后台隔离区单独初始化。

---

## 关闭隔离区

多个客户端可以重复连接同一个 `DriftIsolate`，因此默认情况下，隔离区的生命周期长于单个连接。**单独关闭一个客户端不会停止隔离区**（避免中断其他数据库实例）。

你需要调用 `DriftIsolate.shutdownAll()` 关闭隔离区和所有客户端，释放全部资源。

大多数场景下，你只需要**单客户端连接**，此时在 `connect()` 中设置 `singleClientMode: true`，关闭唯一连接时会自动销毁隔离区。

## 常见工作模式

### 1. 一个执行隔离区 + 一个前台隔离区

这是最常用的模式，在 Flutter 主函数中调用 `DriftIsolate.spawn`，主隔离区通过 `connect` 连接使用。

> 小贴士：该模式与 `NativeDatabase.createInBackground` 效果完全一致，无需手动编写。

### 2. 一个执行隔离区 + 多个客户端隔离区

`DriftIsolate` 可以被传输到多个隔离区，每个隔离区独立调用 `connect`。适用于多线程架构：

- Drift 执行隔离区（核心）
- Flutter 前台 UI 隔离区
- 网络/耗时任务后台隔离区

任意实例的写入操作都会同步到其他隔离区，并自动更新流查询。
接收端可以通过 `DriftIsolate.fromConnectPort` 从端口重建 `DriftIsolate` 实例。

## 工作原理与局限性

### 全功能支持

后台隔离区支持 Drift **所有功能**，开箱即用：

- 事务
- 自动更新流查询（跨隔离区生效）
- 批量增删改
- 自定义语句 / SQL API 生成的语句

### 性能说明

使用后台隔离区可以避免 UI 线程卡顿，但**数据库整体性能会略有下降** —— 跨隔离区传输数据存在性能开销。如果你的应用没有出现UI掉帧问题，无需使用隔离区。不过，新版 Dart/Flutter 大幅优化了隔离区性能。

### 内部架构

1. **服务端隔离区**：执行所有查询、广播表更新，支持任意数量客户端通过类 RPC 模式连接，基于 `SendPort`/`ReceivePort` 通信；
2. **客户端隔离区**：构建 SQL 语句，将语句和参数发送给服务端执行，底层指令的封装让你可以复用所有原有代码。

---

## 独立隔离区共享（进阶）

前文的方案都需要一个主隔离区创建并共享 `DriftIsolate`，但 Flutter 应用中存在特殊场景：

- 后台任务/关闭状态下的 FCM 推送
- 原生平台管理的多个 `FlutterEngine`

这些场景没有明确的隔离区通信方案，但仍需要共享同一个数据库实例。

解决方案：使用 `dart:ui` 的 `IsolateNameServer`，将 `DriftIsolate` 的连接端口注册到全局名称服务，其他客户端可以通过名称获取端口并重建 `DriftIsolate`。

> 注意：Flutter 目前存在一些底层限制：
>
> 1. 后台引擎创建隔离区存在兼容性问题；
> 2. 热重载不会清空名称服务，端口会失效；
> 3. 无法可靠检测端口是否有效。

该模式的实现方案与问题详见官方 Issue。

---

### 核心总结

1. **默认方案**：`NativeDatabase.createInBackground` 自动使用后台隔离区，满足绝大多数场景；
2. **临时任务**：使用 `computeWithDatabase` 跨隔离区执行耗时操作；
3. **手动控制**：通过 `DriftIsolate` 实现多隔离区共享数据库，支持全功能同步；
4. **兼容旧版 Flutter**：主隔离区计算路径，传递给后台隔离区创建数据库。
