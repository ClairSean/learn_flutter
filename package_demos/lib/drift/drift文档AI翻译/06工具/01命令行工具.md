# 命令行工具（CLI）

用于操作 Drift 项目的一套命令行工具

## 实验性功能

Drift 命令行工具目前处于**实验阶段**。如发现任何问题，请及时反馈。

## 使用方法

如果你的项目依赖了 `drift_dev`，即可直接使用该命令行工具。
本文中统一使用 `dart run drift_dev` 启动工具，**必须在项目根目录（`pubspec.yaml` 所在文件夹）**执行命令。

---

## 分析（Analyze）

在项目中所有 `.drift` 文件上运行 Drift 的分析器与代码检查工具。

```bash
dart run drift_dev analyze
```

### 输出示例

```
WARNING: For file test/data/tables/tables.drift:
WARNING: line 38, column 28: This table has columns without default values, so defaults can't be used for insert.
   ╷
38 │ defaultConfig: INSERT INTO config DEFAULT VALUES;
   │                            ^^^^^^
   ╵
INFO: Found 1 errors or problems
```

若检测到任何错误，命令会以**退出码 1**终止。

---

## 识别数据库（Identify databases）

该命令主要用于**测试**，验证 Drift 分析器是否正常工作。
它会扫描并列出项目中所有定义的 Drift 数据库或数据访问对象（DAO）。

```bash
dart run drift_dev identify-databases
```

### 输出示例

```
Starting to scan in /tmp/example/ ...
INFO: example/example.dart has drift databases or daos: Database
INFO: lib/src/data/database.dart has drift databases or daos: AppDatabase
INFO: test/fake_db.dart has drift databases or daos: TodoDb, SomeDao
```

## 数据库结构工具（Schema tools）

### 导出结构用于版本控制（Dump for version control）

该子命令接收两个路径参数：

1. 一个 Dart 文件路径（文件中**必须包含且仅包含一个**被 `@DriftDatabase` 注解的数据库类）
2. 目标输出文件路径

执行命令会将数据库的完整结构导出为 JSON 文件。

```bash
dart run drift_dev schema dump path/to/database.dart schema.json
```

### 导出文件包含的内容

生成的 JSON 文件包含数据库的完整信息：

- 所有数据表（含列的详细配置）
- 触发器
- 索引
- 引入的 Drift 文件中的 `@create-queries` 语句
- 上述元素的依赖关系

**用途**：导出结构文件可用于**生成数据库迁移的测试代码**，详细用法参考迁移测试指南。

---

### 导出 SQL 创建语句（Exporting）

在部分场景中，需要导出定义 Drift 数据库的所有 `CREATE` 语句，该命令可实现此功能。
命令接收一个包含 Drift 数据库定义的 Dart 文件路径作为参数。

```bash
dart run drift_dev schema export path/to/database.dart
```

### 输出说明

命令会输出**全新创建数据库时**，Drift 会自动执行的所有 SQL 语句，**每行一条**。

### 可选参数

`--dialect`：控制生成的 SQL 方言

- 默认值：`sqlite`
- 实验性支持：`postgres`、`mariadb`
