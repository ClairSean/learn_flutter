# 支持的平台

Drift 支持的所有平台及对应使用方法

Drift 基于 sqlite3 数据库构建，几乎可以在所有 Dart 平台上运行。自首次发布以来，Dart 与 Flutter 生态发生了巨大变化。为厘清不同 Drift 包的区别与适用场景，本文列出了全部支持的平台，以及面向各平台的 Drift 使用方案。

为实现平台无关性，Drift 将**核心 API**与**平台专属数据库实现**分离。核心 API 采用纯 Dart 编写，可在所有 Dart 平台运行（甚至脱离 Flutter 环境）。开发 Drift 应用时，建议优先使用 `package:drift/drift.dart` 中的 API，这些 API 保证全平台兼容。你可以根据目标平台选择不同的 `QueryExecutor`（查询执行器）——该接口负责将 Drift 核心库与原生数据库绑定。

---

## 概览

下表列出了 Drift 所有支持的数据库实现，及其适用平台：

| 实现                                                | 支持平台                            | 说明                                                                                                        |
| --------------------------------------------------- | ----------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| `package:drift_sqflite` 中的 `SqfliteQueryExecutor` | Android、iOS                        | 基于平台通道实现，**仅支持 Flutter**；不支持隔离区（Isolate），不兼容 `flutter test`；旧名为 `moor_flutter` |
| `package:drift/native.dart` 中的 `NativeDatabase`   | Android、iOS、Windows、Linux、macOS | 无需额外配置，**推荐在隔离区中使用**；旧名为 `package:moor/ffi.dart`                                        |
| `package:drift/wasm.dart` 中的 `WasmDatabase`       | Web                                 | 兼容 Flutter 与纯 Dart 环境，需少量额外配置                                                                 |
| `package:drift/web.dart` 中的 `WebDatabase`         | Web                                 | 已弃用，推荐使用 `WasmDatabase`                                                                             |

若要在共享代码库中兼容所有平台，**仅需修改数据库的打开方式**，其余业务代码均可保持不变。该代码仓库提供了使用**条件导入**实现跨平台兼容的示例。

## 移动端（Android 和 iOS）

移动端提供两种 Drift 实现方案：

### 使用 drift_sqflite

`drift_sqflite`（旧名 `moor_flutter`）是基于 `sqflite` 包实现的 Drift 数据库适配器。它通过 Flutter 平台通道工作，支持 Android 和 iOS 平台，**无法用于非 Flutter 的纯 Dart 项目**。

对于新项目，我们推荐使用基于 FFI 的新版实现，但 `drift_sqflite` 仍在持续维护并提供支持。

### 使用 drift/native

新版 `package:drift/native.dart` 实现通过 `dart:ffi` 绑定 sqlite3 原生 C API。这是入门指南中推荐的**新项目首选方案**。

自 Drift 依赖的 `sqlite3` 包 **3.0.0 版本**起，新版 SQLite 会自动打包进你的 Flutter 应用。关于该机制的更多信息与配置方式，可查阅官方文档。

#### 关于 FFI 与 Android 的说明

`package:drift/native.dart` 是新版 Android 应用的推荐 Drift 实现。但部分设备上存在以下小问题：

- **复杂查询触发内存溢出**：由于 Android 无法使用默认临时文件目录，需要告知 sqlite3 正确的临时数据存储路径。可参考官方示例实现配置。

## 网页端

主文章：[网页端支持](https://drift.simonbinder.eu/docs/platforms/web/)

Drift 通过将 sqlite3 编译为 **WebAssembly 模块**实现网页端兼容，可通过 `package:drift/wasm.dart` 中的 `WasmDatabase` 访问数据库。为最大化兼容不同浏览器，需要配置工作脚本并完成少量额外设置。主文章详细说明了 Drift 网页端的完整配置流程。

---

## 桌面端

Drift 通过 `package:drift/native.dart` 中的 `NativeDatabase`，支持所有运行 Dart 的主流桌面操作系统。

自 Drift **2.32.0 版本**（依赖 3.x 版本的 `sqlite3` 包）起，**无需额外配置**，最新版 SQLite 会自动打包进应用。更多信息可查看 `sqlite3` 构建钩子文档。

无需再依赖 `sqlite3_flutter_libs` 或 `sqlcipher_flutter_libs`，可直接移除这些依赖项。
