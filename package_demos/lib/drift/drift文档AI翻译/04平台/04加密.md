# 加密

在加密数据库中使用 Drift

在 Drift 中使用加密数据库有两种方式。`encrypted_drift` 包与 `drift_sqflite` 类似，基于 Java 编写的平台插件实现。另一种方式是使用标准的 `package:drift/native.dart` API，搭配基于 FFI 的实现。

**对于新应用，我们推荐使用 `NativeDatabase` 搭配加密数据库**。使用新版加密包的 Flutter 应用示例可在官方仓库中查看。

---

## 加密版 NativeDatabase

你可以将 Drift 原生库与加密执行器结合使用，这能让你在**所有原生平台（包含桌面端）**上使用加密的 Drift 数据库。

### 配置

`sqlite3` 包通过构建钩子将指定版本的 SQLite 打包进你的应用。我们可以将其配置为打包 **SQLite3MultipleCiphers**（提供加密支持的 SQLite 分支）。

如需启用加密，将以下配置添加到**工作区根目录的 `pubspec.yaml`** 中（若未使用工作区，则添加到应用的 `pubspec.yaml`）：

```yaml
hooks:
  user_defines:
    sqlite3:
      source: sqlite3mc
```

关于构建钩子的更多配置说明，请查阅官方文档。

旧版 `sqlite3` 包需要依赖 `sqlcipher_flutter_libs`。升级到 Drift 2.32.0 后，该依赖已不再需要，可以直接移除。

### 使用方法

SQLite3MultipleCiphers 完全兼容 SQLite 的 C 语言 API，这意味着你可以继续使用 `sqlite3` 包或 `package:drift/native.dart`，**无需修改现有代码**。

若要对数据库进行加密，必须在使用数据库**之前**设置加密密钥。在 Drift 中，最佳实践是在 `NativeDatabase` 的 `setup` 参数中配置，该回调会在 Drift 操作数据库前执行：

```dart
NativeDatabase.createInBackground(
  myDatabaseFile,
  setup: (rawDb) {
    // 设置数据库加密密钥
    rawDb.execute("PRAGMA key = 'passphrase';");
  },
);
```

### 重要提示

SQLite3MultipleCiphers 是集成了加密功能的 SQLite 分支。为确保应用中使用了正确的加密版本，建议在运行时校验加密指令是否可用（原生 SQLite 不提供该指令）：

```dart
bool _debugCheckHasCipher(Database database) {
  return database.select('PRAGMA cipher;').isNotEmpty;
}
```

在使用数据库前，添加断言校验：

```dart
NativeDatabase.createInBackground(
  myDatabaseFile,
  setup: (rawDb) {
    // 校验加密支持
    assert(_debugCheckHasCipher(rawDb));
    rawDb.execute("PRAGMA key = 'passphrase';");
  },
);
```

如果校验失败，请检查构建钩子配置，或在官方仓库提交 Issue 寻求帮助。

## 加密现有数据库

若你需要为已有的普通数据库添加加密，需遵循以下步骤：

1. 按照上文配置，启用 SQLite3MultipleCiphers；
2. **注意**：不能直接对现有数据库执行 `key` 指令！

正确的迁移方式是：创建临时副本，通过 `rekey` 指令完成加密：

```dart
final existingDatabasePath = '/path/to/your/database.db';
final encryptedDatabasePath = '/path/to/your/encrypted.db';
const yourKey = 'passphrase';

// 转义字符串，避免SQL注入
String escapeString(String source) {
  return source.replaceAll('\'', '\'\'');
}

// 加密数据库实例
NativeDatabase.createInBackground(
  File(encryptedDatabasePath),
  isolateSetup: () async {
    final existing = File(existingDatabasePath);
    final encrypted = File(encryptedDatabasePath);

    // 原有数据库存在，且加密数据库未创建时执行迁移
    if (await existing.exists() && !await encrypted.exists()) {
      final tmp = File('${encrypted.path}.tmp');
      if (await tmp.exists()) await tmp.delete();

      // 将未加密数据库导出到临时文件
      final plaintextDb = sqlite3.open(existingDatabasePath)
        ..execute("VACUUM INTO '${escapeString(tmp.path)}';");
      plaintextDb.close();

      // 为临时文件设置加密密钥
      final encryptedDb = sqlite3.open(tmp.path);
      encryptedDb.execute("PRAGMA rekey = '${escapeString(yourKey)}';");
      encryptedDb.close();

      // 重命名为加密数据库，删除原文件
      await tmp.rename(encryptedDatabasePath);
      await existing.delete();
    }
  },
  setup: (rawDb) {
    assert(_debugCheckHasCipher(rawDb));
    rawDb.execute("PRAGMA key = '${escapeString(yourKey)}';");
  },
);
```

## 从 SQLCipher 迁移到 SQLite3MultipleCiphers

本文旧版推荐使用 `sqlcipher_flutter_libs` 和自定义代码启用 SQLCipher 加密。
从 Drift 2.32.0 + `sqlite3` 3.x 版本开始，使用 SQLite3MultipleCiphers 更加简便。

SQLCipher 已不再支持简易配置，但**现有 SQLCipher 加密的数据库可以兼容 SQLite3MultipleCiphers**。
升级后，在设置密钥前执行以下两条指令即可兼容：

```sql
PRAGMA cipher = 'sqlcipher';
PRAGMA legacy = 4;
```

升级后请严格测试应用对旧加密数据库的兼容性，详情查看官方 Issue。

## 使用 encrypted_drift 包

除了基于 `dart:ffi` 和 `sqlite3` 的加密执行器，我们还提供了基于 `sqflite_sqlcipher` 库（作者：@davidmartos96）的 Drift 加密版本。

使用方式：

1. 从 `pubspec.yaml` 中移除 `drift_sqflite` 依赖，替换为：

```yaml
dependencies:
  drift: ^2.32.1
  encrypted_drift:
    git:
      url: https://github.com/simolus3/drift.git
      path: extras/encryption
```

2. 将原有导入的 `package:drift_sqflite/drift_sqflite.dart` 或 `package:drift/native.dart`，替换为：

```dart
import 'package:drift/drift.dart';
import 'package:encrypted_drift/encrypted_drift.dart';
```

3. 将 `SqfliteQueryExecutor` 或 `NativeDatabase` 替换为 `EncryptedExecutor`。

### Android 和 iOS 额外配置

为保证 SQLCipher 正常工作，项目中可能需要额外步骤，例如：

- Android 正式版应用的 ProGuard 混淆配置

你可以查阅库的官方说明（无需关注库的安装与使用步骤，`encrypted_drift` 已内部处理）。
