# 代码生成配置选项

用于配置 `drift_dev` 和 `build_runner`、自定义 Drift 生成代码规则的全部选项

drift_dev 包提供了大量可配置项，用于控制代码生成逻辑。大多数场景下**默认配置**即可满足需求。如果你想要更快体验新功能、自定义 Drift 生成代码的结构与样式，可以使用本文列出的全部配置。文末也提供了推荐配置方案，告诉你哪些选项适合开启/关闭。

## 使用配置

在项目根目录（与 `pubspec.yaml` 同级）新建 `build.yaml` 文件即可生效：

```yaml
# build.yaml 配置功能强大，详见 https://pub.dev/packages/build_config
targets:
  $default:
    builders:
      drift_dev:
        options:
          store_date_time_values_as_text: true
```

## 全部可用配置项

当前 Drift 支持以下生成配置：

1. **write_from_json_string_constructor**（布尔值）
   为生成的数据类新增 `.fromJsonString` 工厂构造方法。
   默认只会生成接收 `Map<String, dynamic>` 的 `.fromJson` 构造方法。

2. **override_hash_and_equals_in_result_sets**（布尔值）
   当 Drift 为自定义查询结果生成独立结果类时，控制是否重写类的 `==` 运算符、`hashCode`。
   开启后，新版 Drift 还会同时重写 `toString` 方法。

3. **skip_verification_code**（布尔值）
   生成的表默认包含大量代码，用于校验插入数据合法性、数据违规时抛出详细错误提示。
   如果你只使用原生 SQL 插入数据、或者不需要数据校验功能，开启此项可以**大幅减少生成代码体积**。

4. **use_data_class_name_for_companions**（布尔值）
   默认：伴随类名称基于表名生成（例如 `@DataClassName('Users')` 的 `UsersTable` 表，会生成 `UsersTableCompanion`）。
   开启此项：伴随类名称基于数据类名生成（上例将生成 `UsersCompanion`）。

5. **use_column_name_as_json_key_when_defined_in_moor_file**（布尔值，默认 `true`）
   针对 `.drift` 文件中定义的列，JSON 序列化/反序列化时**使用 SQL 原始列名**，而非 Dart getter 属性名。
   例如 SQL 列 `user_name` 序列化后键名为 `user_name`，而不是 `userName`。
   你可以通过 `JSON KEY` 列约束单独覆盖某一列的 JSON 键名：
   `user_name VARCHAR NOT NULL JSON KEY userName`。

6. **use_sql_column_name_as_json_key**（布尔值，默认 `false`）
   无论表是 Dart 定义还是 drift 文件定义，全部统一使用 **SQL 原始列名**作为 JSON 序列化键名。

7. **generate_connect_constructor**（已废弃）
   在数据库类生成命名构造方法 `connect()`，接收 `DatabaseConnection` 而非 `QueryExecutor`。
   Drift 2.5 已废弃该选项，因为 `DatabaseConnection` 现已实现 `QueryExecutor` 接口。

8. **data_class_to_companions**（布尔值，默认 `true`）
   控制是否在生成的数据类中编写 `toCompanion` 转换方法。

9. **mutable_classes**（布尔值，默认 `false`）
   默认：数据类、伴随类、查询结果类的字段全部是 `final` 不可变。
   开启 `mutable_classes: true` 后，字段变为可变（非 final）。

10. **row_class_constructor_all_required**（布尔值，默认 `false`）
    生成的行类（表类、自定义查询结果类）所有构造参数强制为**必传**，无论字段是否可空。
    因为这类类永远代表数据库完整一行，强制必传可以更贴合语义。

11. **raw_result_set_data**（布尔值）
    生成的查询结果类中暴露底层原始 `QueryRow` 数据。

12. **apply_converters_on_variables**（布尔值，默认 `true`）
    对编译后 SQL 语句中的变量自动应用类型转换器。

13. **generate_values_in_copy_with**（布尔值，默认 `true`）
    在 `copyWith` 方法中，可空列使用 `Value<T?>` 而非 `T?`。
    开启后可以通过 `Value(null)` 将字段显式置空；
    默认直接传 `null` 会被忽略，无法将字段清空。

14. **named_parameters**（布尔值）
    为 SQL 查询中的命名变量，生成 Dart 命名参数。

15. **named_parameters_always_required**（布尔值）
    所有命名参数（开启 `named_parameters` 后生成）强制为 Dart 必传参数。

16. **scoped_dart_components**（布尔值，默认 `true`）
    为 SQL 中的 Dart 占位符生成函数类型参数，函数内置当前查询可用的所有表参数，方便编写表别名。

17. **store_date_time_values_as_text**（布尔值）
    日期时间列是否以 **ISO 8601 字符串**存储，而非 Unix 时间戳。详见[日期时间配置文档](https://drift.simonbinder.eu/docs/reference/datetime-options)。

18. **case_from_dart_to_sql**（默认 `snake_case`）
    控制 Dart 标识符转换为 SQL 表名/列名的大小写规则。
    可选值：`preserve`（保持原样）、`camelCase`、`CONSTANT_CASE`、`snake_case`（蛇形小写，默认）、`PascalCase`、`lowercase`、`UPPERCASE`。

19. **write_to_columns_mixins**（布尔值）
    控制 `toColumns` 方法是否以 Mixin 形式生成，而不是直接写在数据类中。
    在使用自定义行类时非常有用，Mixin 也会为自定义行类生成。

20. **has_separate_analyzer**（布尔值）
    仅在使用 `drift_dev:not_shared` 构建器时生效。该构建器默认使用低效分析实现，同目标开启 `drift_dev:analyzer` 后打开此项可**加速构建**。
    默认构建器、模块化构建器对此选项无效。

21. **fatal_warnings**（布尔值，默认 `false`）
    开启后，drift_dev 在构建阶段检测到警告（SQL语法错误、Dart表引用未找到等）会直接**构建失败**，而非仅警告。

22. **preamble**（字符串）
    独立 part 构建器、模块化构建专用配置。原生 source_gen 的 preamble 配置在此环境无效，因此 Drift 单独提供该配置。

23. **generate_manager**（布尔值，默认 `true`）
    开启后为数据库每张表自动生成 **Manager 管理器类**，无需手写模板代码即可完成简单增删改查。

## 预设 SQL 运行环境

你可以通过 `sql` 构建配置指定目标 SQL 方言；使用 SQLite 时，还可以预设 SQLite 版本、已启用扩展，让静态分析更精准。

> 注意：这些选项仅用于**编译期静态分析**，不会修改运行时真实 SQLite 版本。

指定 SQLite 版本示例：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          sql:
            dialect: sqlite
            options:
              version: "3.34"
```

配置后，使用高于该版本的 SQLite 新特性时，生成器会抛出警告。
例如 SQLite 3.34 不支持多条 upsert 子句，使用会报错。
当前最低支持分析版本为 3.34。

## 多数据库方言代码生成

得益于社区贡献，Drift 现已支持 **Postgres、MariaDB**，将方言改为对应名称即可生成对应数据库代码。

部分场景需要一套代码兼容多种数据库（后端 Flutter 共用、服务端同时支持 MariaDB/Postgres），Drift 支持**多方言同时生成**，运行时自动选择对应 SQL。

配置方式：删除单个 `dialect`，改为方言列表 `dialects`：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          sql:
            dialects:
              - sqlite
              - postgres
            options:
              version: "3.34"
```

## 支持的数据库扩展

> 注意：此处仅开启**分析器支持**（drift 文件可编写对应函数语法），不代表运行时一定支持！
> iOS 11+ 原生支持；Android 仅在使用 `NativeDatabase` 时支持。

配置示例：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          sql:
            dialect: sqlite
            options:
              modules:
                - json1
                - fts5
                - math
                - dbstat
```

当前支持扩展列表：

1. **json1**：支持 drift 文件中所有 json\_\* 函数静态分析
2. **fts5**：支持 FTS5 虚拟表、MATCH 检索运算符，支持 highlight、bm25 等函数
3. **rtree**：R\*Tree 空间索引扩展静态分析；sqlite3 包默认已编译开启，安全可用
4. **geopoly**：R\*Tree 升级版，支持复杂多边形；大部分 SQLite 编译版不自带
5. **dbstat**：统计表磁盘占用空间；需要 SQLite 编译标记，sqlite3 包默认开启
6. **moor_ffi**：仅 NativeDatabase 支持的扩展函数（pow、sqrt、三角函数等）
7. **math**：数学函数扩展；与 moor_ffi 大部分不兼容
8. **spellfix1**：拼写纠错扩展；大部分 SQLite 编译版不自带
9. **powersync**：PowerSync 同步 SQLite 扩展，支持建表时引用 powersync_crud

## 自定义已知 SQL 函数

`modules` 用于声明官方扩展；你也可以自定义**原生 SQL 函数**，告知分析器函数签名，即可在 .drift 文件中正常使用。

在配置中添加 `known_functions`：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          sql:
            dialect: sqlite
            options:
              known_functions:
                my_function: "boolean (text, int null)"
```

含义：存在 SQL 函数 `my_function`，接收**非空文本、可空整数**，返回非空布尔值，Drift 可正常类型推断。

函数类型语法：
`<返回类型>(<参数类型>)`
类型可使用任意关键字，Drift 支持 `DATETIME`、`BOOLEAN` 类型提示；末尾加 `NULL` 表示可空。

## 外部数据表

除自定义函数外，你还可以声明**不受 Drift 管理、数据库中已存在的外部表**，避免分析器抛出未定义警告。
Drift 不会为外部表生成任何代码。

```yaml
targets:
  $default:
    builders:
      drift_dev:
        options:
          sql:
            dialect: sqlite
            options:
              known_tables:
                - "CREATE TABLE external_resource (id INTEGER PRIMARY KEY, another TEXT)"
```

## 推荐优化配置（精简生成代码）

官方默认配置最完整，但你可以关闭部分功能减小代码体积：

1. `skip_verification_code: true`：大幅精简生成代码，代价是插入非法数据时错误提示不详细
2. `data_class_to_companions: false`：不需要 `toCompanion` 方法时关闭
