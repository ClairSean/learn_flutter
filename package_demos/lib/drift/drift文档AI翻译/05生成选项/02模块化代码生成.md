# 模块化代码生成

让 Drift 将代码生成到多个文件中

默认情况下，Drift 从**单个入口文件**生成代码：数据库的所有表、视图和查询语句都会被生成到同一个 `part` 文件中。对于大型项目，这个文件会变得非常庞大，重新生成时会降低构建速度和代码分析器的效率。

Drift 提供了一种可选的**模块化代码生成模式**，专为大型项目设计。通过这种配置，Drift 会生成多个文件，并自动管理文件之间的导入关系。

---

举个直观的例子：假设一个大型 Drift 项目包含大量表和视图，拆分到了不同的文件中：

```
lib/src/database/
├── database.dart
├── tables/
│   ├── users.drift
│   ├── settings.drift
│   ├── groups.drift
│   └── search.drift
└── views/
    ├── friends.drift
    └── categories.dart
```

虽然这种模块化结构（在 Drift 文件中使用导入）有助于组织源码，但默认模式下，Drift 仍然会把所有代码生成到**单个** `database.g.dart` 文件中。随着表和查询语句的增多，Drift 可能需要为数据类、伴随类和查询结果生成数万行代码。

启用**模块化生成模式**后，Drift 会为**每个输入文件单独生成源码**，结构如下：

```
lib/src/database/
├── database.dart
├── database.drift.dart
├── tables/
│   ├── users.drift
│   ├── users.drift.dart
│   ├── settings.drift
│   ├── settings.drift.dart
│   └── ...
└── views/
    ├── friends.drift
    ├── friends.drift.dart
    ├── categories.dart
    └── categories.drift.dart
```

## 启用模块化代码生成

注意：Drift 仓库中包含一个使用模块化代码生成的小型示例。

由于 Drift 的模块化代码生成模式与默认构建器生成的文件规则不同，**需要显式启用**。
你需要创建一个 `build.yaml` 文件：禁用默认的 `drift_dev` 构建器，手动启用两个模块化构建器：`drift_dev:analyzer` 和 `drift_dev:modular`。两个构建器需要配置**完全相同**的参数：

```yaml
targets:
  $default:
    builders:
      drift_dev:
        # 禁用 Drift 默认构建器，改用模块化模式
        enabled: false

      # 手动启用模块化构建器
      drift_dev:analyzer:
        enabled: true
        options: &options # Drift 构建配置，参考官方文档
          store_date_time_values_as_text: true
          named_parameters: true
          sql:
            dialect: sqlite
            options:
              version: "3.39"
              modules: [fts5]
      drift_dev:modular:
        enabled: true
        # 使用 YAML 锚点保证两个构建器配置一致
        options: *options
```

## 生成的文件内容

启用模块化生成后，Drift 会生成**独立的 Dart 库**（不再包含 `part of` 语句）。
这也意味着：你**不再需要**在源码中编写 `part` 语句，直接导入生成的 `.drift.dart` 文件即可。

同时，之前在 `part` 文件中生成的所有私有混入/类，现在都会变为公开的。
将代码中所有 `_$` 替换为 `$` 即可修复相关命名问题。

在代码使用上，大部分逻辑没有变化：数据库和数据访问对象（DAO）的 API 基本保持一致。
**唯一的重大区别**是 `.drift` 文件的处理方式：

- 默认模式：所有 Drift 文件中的查询都会生成为**数据库类的方法**；
- 模块化模式：Drift 会生成**隐式数据库访问器**，通过数据库类的 getter 方法调用。

举个例子，`user.drift` 文件：

```sql
CREATE TABLE users (
  id INTEGER NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  name TEXT NOT NULL,
  is_admin BOOLEAN NOT NULL DEFAULT FALSE
);

findUsers($predicate = TRUE): SELECT * FROM users WHERE $predicate;
```

在模块化模式下，Drift **不会**为数据库类生成 `findUsers` 方法。
取而代之，`users.drift.dart` 文件中会生成一个名为 `UsersDrift` 的数据库访问器，并自动挂载到数据库类上。

调用方式变更为：

```dart
// 旧写法（默认模式）
database.findUsers();

// 新写法（模块化模式）
database.usersDrift.findUsers();
```
