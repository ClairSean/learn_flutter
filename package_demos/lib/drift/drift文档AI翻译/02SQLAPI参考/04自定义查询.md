# 自定义查询

为目前 Drift 暂无法通过 Dart 实现的高级查询编写原生 SQL

尽管 Drift 提供了一套流畅的 API 来构建绝大多数数据库语句，但**WITH 子句**、部分子查询等高级功能暂不支持。不过你可以通过 `customSelect`、`customStatement` 等方法手动编写 SQL，执行这些高级语句。

对于大多数自定义查询，Drift 会在**编译时**分析 SQL 语法、校验合法性，并为其生成**类型安全的 Dart API**。这种方式比在运行时编写原生 SQL 更安全可靠。

本文将介绍两种实现方式：第一部分讲解 Drift 自动生成的查询方法，第二部分展示运行时定义自定义查询的示例。

---

## 带自动生成 API 的查询语句

你可以让 Drift 为 `SELECT`、`UPDATE`、`DELETE` 语句**自动生成类型安全的 API**，当然你也可以选择手动编写原生 SQL。具体用法如下：

只需在 `@DriftDatabase` 注解中定义你的查询语句，即可启用该功能：

```dart
@DriftDatabase(
  tables: [TodoItems, Categories],
  queries: {
    'categoriesWithCount':
        'SELECT *, '
        '(SELECT COUNT(*) FROM todo_items WHERE category = c.id) AS "amount" '
        'FROM categories c;',
  },
)
class MyDatabase extends $MyDatabase {
  // 类的其余代码保持不变
}
```

重新执行代码构建后，Drift 会为你生成 `CategoriesWithCountResult` 类（用于承载查询结果）。同时，你继承的 `_$MyDatabase` 父类会新增一个 `Selectable<CategoriesWithCountResult> categoriesWithCount()` 方法，用于执行该查询。

和 Drift 中所有的可查询对象一样，你可以使用 `get()` 单次执行查询，或使用 `watch()` 获取自动更新的结果流：

### Drift 文件中对自定义查询的优化支持

在 `@DriftDatabase` 注解中定义 SQL 适合少量自定义查询。如果你的应用包含大量自定义查询，将其抽离到独立文件中会更易于管理。**可直接集成到数据库中的 Drift 文件**是最佳选择，使用起来也更便捷。

```dart
Future<void> useGeneratedQuery() async {
  // 生成的查询可以作为 Future 单次执行：
  await categoriesWithCount().get();

  // 也可以作为流多次监听，自动更新数据
  await for (final snapshot in categoriesWithCount().watch()) {
    print('查询到 ${snapshot.length} 个分类结果');
  }
}
```

查询可以通过 `?` 或 `:变量名` 语法传入参数。Drift 会自动推断参数的合适类型，并将其加入生成的方法中。
例如：`'categoryById': 'SELECT * FROM categories WHERE id = :id'` 会生成方法 `categoryById(int id)`。

Drift 还为自定义查询提供了更多便捷功能，比如在 SQL 中嵌入 Dart 表达式。更多详情请参考 Drift 文件的相关文档。

---

### 关于表名

使用该功能时，你需要了解 Dart 定义的表在 SQL 中的命名规则：

- 未重写 `tableName` 的表，SQL 中的表名为**类名的蛇形命名（snake_case）**
  例如：Dart 表 `Categories` → SQL 表名 `categories`；`UserAddressInformation` → `user_address_information`
- 未显式命名的列 getter 也遵循此规则
- Drift 文件中声明的表和列，始终使用你指定的名称

你也可以用这种方式定义 `UPDATE` 或 `DELETE` 语句。该功能同样适用于 DAO，并且 Drift 会通过分析查询读写的表，完美兼容自动更新流。

## 自定义查询语句

如果你不想使用自动生成 API 的查询，也可以直接调用方法执行自定义查询：

- `customSelect`：执行单次查询
- `customSelectStream`：执行查询流，底层数据变化时自动触发更新

以下是入门指南中的待办事项示例，我们编写一个查询，加载每个分类下的待办事项数量：

```dart
// 自定义结果类
class CategoryWithCount {
  final Category category;
  final int count; // 当前分类下的待办事项数量

  CategoryWithCount({required this.category, required this.count});
}

// 在数据库类中定义方法
Stream<List<CategoryWithCount>> allCategoriesWithCount() {
  return customSelect(
    'SELECT *, (SELECT COUNT(*) FROM todos WHERE category = c.id) AS "amount"'
    ' FROM categories c;',
    // 声明查询依赖的表：当这些表发生变化时，流会自动更新
    readsFrom: {todoItems, categories},
  ).watch().map((rows) {
    // 将原始查询结果转换为自定义对象
    return rows
        .map(
          (row) => CategoryWithCount(
            category: categories.map(row.data), // Drift 已知如何解析分类数据
            count: row.read<int>('amount'), // 手动读取统计数量
          ),
        )
        .toList();
  });
}
```

编写自定义查询时，建议通过 `readsFrom` 参数指定查询读取的表。使用流时，Drift 才能判断何时需要触发流更新。

---

你也可以使用**问号占位符**配合 `variables` 参数绑定 SQL 变量：

```dart
Stream<int> amountOfTodosInCategory(int id) {
  return customSelect(
    'SELECT COUNT(*) AS c FROM todo_items WHERE category = ?',
    variables: [Variable.withInt(id)],
    readsFrom: {todoItems},
  ).map((row) => row.read<int>('c')).watchSingle();
}
```

当然，你也可以使用索引变量（如 `?12`），更多信息请查阅 SQLite3 官方文档。

---

## 自定义更新语句

对于 `UPDATE` 和 `DELETE` 语句，你可以使用 `customUpdate` 方法。和 `customSelect` 一样，该方法支持传入 SQL 语句和可选的变量参数。

你可以通过可选参数 `updates` 告知 Drift 该查询会影响哪些表，这会帮助其他查询流实现自动更新。
