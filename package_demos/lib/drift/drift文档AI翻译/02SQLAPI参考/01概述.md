# 验证式 SQL

在不丢失**类型安全**的前提下，使用 SQL 定义数据库结构与查询语句

Drift 提供了 Dart API 用于定义数据表、编写 SQL 查询。如果你已经熟悉 SQL，直接通过 `CREATE TABLE` 语句定义数据表会更加便捷。
得益于 Drift 内置的强大 SQL 解析器与分析器，你依然可以执行**类型安全**的 SQL 查询，同时支持自动更新流、事务等所有 Drift 核心功能。
你的 SQL 语法有效性会在**构建阶段**被校验，Drift 会自动为每张表、每条查询语句生成配套的 Dart 方法。

---

## 设置

Drift 依赖的基础配置与 Dart API 模式完全一致，详细步骤请参考[安装页面](https://drift.simonbinder.eu/docs/getting-started/)。

两种模式的核心区别在于**数据表和查询的声明方式**：
想要让 Drift 识别 SQL 代码，需要将其写入 **`.drift`** 后缀的文件中。
在本示例中，我们在数据库类的同级目录下创建一个名为 `tables.drift` 的文件：

```sql
-- 这是 tables.drift 文件
CREATE TABLE todos (
    id INT NOT NULL PRIMARY KEY AUTOINCREMENT,
    title TEXT,
    body TEXT,
    category INT REFERENCES categories (id)
);

CREATE TABLE categories (
    id INT NOT NULL PRIMARY KEY AUTOINCREMENT,
    description TEXT
) AS Category; -- 下方有 AS Category 的详细说明

/* 声明完表后，可以直接在这里编写查询语句
   格式：查询名称 :  SQL 语句 */
todosInCategory: SELECT * FROM todos WHERE category = ?;

/* 这是一个更复杂的查询：统计每个分类下的条目数量
   包含未归属任何分类的条目 */
countEntries:
  SELECT
    c.description,
    (SELECT COUNT(*) FROM todos WHERE category = c.id) AS amount
  FROM categories c
  UNION ALL
  SELECT null, (SELECT COUNT(*) FROM todos WHERE category IS NULL);
```

### 关于 `AS Category` 语法

Drift 会为数据表自动生成对应的 Dart 数据类，类名默认根据表名生成：**自动去除表名末尾的 `s`**。
这个规则适用于大多数场景，但在 `categories` 这类表上会失效（默认会生成 `Categorie`，而非正确的 `Category`）。

因此我们可以在表定义末尾添加 `AS <类名>`，手动指定 Drift 生成的 Dart 类名称。

## 集成 .drift 文件到数据库

将 `.drift` 文件集成到数据库非常简单，只需把文件路径添加到 `@DriftDatabase` 注解的 `include` 参数中即可。
由于我们使用 SQL 定义表，因此可以省略 `tables` 参数。

```dart
import 'dart:io';
import 'package:drift/drift.dart';
// 以下导入用于打开数据库连接
import 'package:drift/native.dart';
import 'package:path_provider/path_provider.dart';
import 'package:path/path.dart' as p;

part 'database.g.dart';

@DriftDatabase(
  // 填入 drift 文件的相对路径（Drift 也支持 package: 格式的导入）
  include: {'tables.drift'},
)
class AppDb extends _$AppDb {
  AppDb() : super(_openConnection());

  // 数据库版本号
  @override
  int get schemaVersion => 1;
}

// 打开数据库连接
LazyDatabase _openConnection() {
  // LazyDatabase 工具可以异步获取数据库文件的正确存储路径
  return LazyDatabase(() async {
    // 将数据库文件 db.sqlite 存放到应用的文档目录中
    final dbFolder = await getApplicationDocumentsDirectory();
    final file = File(p.join(dbFolder.path, 'db.sqlite'));

    return NativeDatabase.createInBackground(file);
  });
}
```

## 生成代码
